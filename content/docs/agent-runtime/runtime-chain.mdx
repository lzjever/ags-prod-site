# Notebook + Files + Agents 运行链路

一条成熟的通用智能体任务链路，不应该依赖临时脚本和个人目录去拼起来。更理想的方式是: 先创建 Notebook Task，再从 Files、URL 或历史 artifacts 中补充输入，随后选择 Agent 与 Endpoint，由系统完成权限与治理预检，准备 Task Workspace 与运行资产，并在执行过程中持续回传 conversation、trace 和 artifacts。任务结束后，结果会继续沉淀为项目文件和任务历史，而不是随着一次运行结束就消失。

  title="这条链路如何工作"
  description="运行前、运行中和运行后的关键对象都会围绕同一个任务持续组织。"

输入不再依赖用户手工拼接目录，运行过程也不再依赖终端窗口单独保存，而结果会继续留在项目上下文中，成为后续可以查看、复用和协作的资产。
