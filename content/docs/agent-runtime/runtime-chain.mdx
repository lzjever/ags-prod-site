# Notebook + Files + Agents 运行链路

理想中的通用智能体任务链路应当是：

1. 创建 Notebook Task。
2. 从 Files / URL / 历史 artifacts 添加输入。
3. 选择 Agent 与 Endpoint。
4. 系统完成权限与治理预检。
5. 准备 Task Workspace 与运行资产。
6. 回传 conversation / trace / artifacts。
7. 把结果沉淀为项目文件与任务历史。

<SiteSection
  title="为什么这条链路重要"
  description="它把运行前、运行中、运行后的关键对象统一组织到一个可回看、可复用的任务对象里。"
>

1. 输入不再依赖用户手工拼目录。
2. 运行过程不再依赖终端窗口。
3. 结果不会随着一次运行结束就消失。

</SiteSection>
