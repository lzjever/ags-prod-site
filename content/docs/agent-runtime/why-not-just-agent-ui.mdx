# 为什么 AgentSmith 不只是一个 Agent UI

很多平台会说自己“支持 Agent”，但它们真正提供的往往只是一个调用方式。

  eyebrow="Agent Runtime"

AgentSmith 的差异化在于，它试图把通用智能体从：

1. 难配置的本地命令行工具
2. 依赖临时目录和脚本的个人工作流
3. 难以沉淀结果和协作的执行环境

升级为：

1. 项目级统一任务空间
2. 可治理、可回看、可复用的执行链路
3. 可逐步走向托管执行与受控环境的平台能力

<SiteSection title="这条差异化由什么构成" description="不是单点页面，而是一组协同对象。">

1. Notebook 任务界面。
2. Files 文件库与挂载桥接。
3. Agents 执行能力接入。
4. Credential Files 与 Skills 注入。
5. Trace / Artifacts / Audit 等运行信息回流。

</SiteSection>
