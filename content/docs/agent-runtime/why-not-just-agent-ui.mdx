# 为什么 AgentSmith 不只是一个 Agent UI

很多平台会说自己“支持 Agent”，但它们真正提供的往往只是一个调用方式。

AgentSmith 的差异化在于，它试图把通用智能体从：

难配置的本地命令行工具、依赖临时目录和脚本的个人工作流，以及难以沉淀结果和协作的执行环境，升级成项目级统一任务空间、可治理可回看可复用的执行链路，以及可逐步走向托管执行与受控环境的平台能力。

<SiteSection title="这条差异化由什么构成" description="不是单点页面，而是一组协同对象。">

这种差异化并不是靠某一个页面撑起来的，而是由一组相互协同的对象共同构成: Notebook 负责承接任务本身，Files 负责文件资产和本地挂载桥接，Agents 负责执行能力接入，Credential Files 与 Skills 负责补足运行所需的上下文，而 Trace、Artifacts、Audit 等运行信息则持续回流，让整个过程始终可被理解和回看。

</SiteSection>
