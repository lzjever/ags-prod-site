# 关键任务流

AgentSmith 的主任务流可以概括为：

1. 系统管理员开通 Workspace。
2. 工作区成员创建并进入 Project。
3. 团队在项目中使用 Chat / Notebook / Files / Agents。
4. 平台形成 Usage / Audit / Trace / Artifacts。
5. 管理员在治理面完成审查、限制和追溯。

<SiteSection
  eyebrow="Differentiator"
  title="通用智能体运行流是当前最值得强调的一条任务流"
  description="这条链路把 Notebook、Files、Agents、Artifacts 和治理预检组织成统一流程。"
>

1. 创建 Notebook Task。
2. 添加项目文件、URL 或历史 artifacts。
3. 绑定 Agent 与 Endpoint。
4. 系统完成权限与治理预检。
5. 执行回传 conversation / trace / artifacts。
6. 结果沉淀为项目资产与后续任务输入。

</SiteSection>
