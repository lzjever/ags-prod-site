# 关键任务流

AgentSmith 的主任务流，从来都不是一组彼此独立的页面跳转，而是一条从工作区开通到项目协作、再到治理回看的连续路径。系统管理员先开通 Workspace，团队成员随后创建并进入 Project，在其中使用 Chat、Notebook、Files 和 Agents，平台则持续形成 Usage、Audit、Trace 和 Artifacts 这些可回看的事实记录，管理员最终再基于这些记录完成限制、审查和追溯。

<SiteSection
  eyebrow="Differentiator"
  title="通用智能体运行流是当前最值得强调的一条任务流"
  description="这条链路把 Notebook、Files、Agents、Artifacts 和治理预检组织成统一流程。"
>

真正体现 AgentSmith 差异化的，是通用智能体运行这条链路。用户会先创建 Notebook Task，再把项目文件、URL 或历史 artifacts 带入上下文，随后绑定 Agent 与 Endpoint，由系统完成权限和治理预检。执行过程中，conversation、trace 与 artifacts 会持续回传，而最终结果又会沉淀为项目资产和后续任务的输入。这条链路清楚说明了 AgentSmith 为什么不是单点调用工具，而是一套面向团队长期运作的系统。

</SiteSection>
