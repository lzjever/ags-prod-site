# Agents

Agents 模块负责让外部或内部智能体以标准协议接入项目。

<SiteSection title="产品定位" description="它不是资源清单页，而是项目级执行能力接入面。">

Agents 模块的意义，不只是列出有哪些智能体，而是把执行能力正式接入项目体系。无论是 external agent 还是 internal agent，都会在这里以统一方式被管理，并通过同一套协议把执行请求、trace 事件和产物回传串起来，成为 Chat 与 Notebook 可以持续调用的能力底座。

</SiteSection>

<SiteSection title="长期价值" description="智能体能力可以在团队中被持续发现、调用和管理。">

Agents 会把原本偏个人化、偏脚本化的能力，升级成团队可发现、可调用、也可治理的项目对象。用户不再需要依赖某个人的本地命令行环境，而可以在项目中直接选择和使用相应执行能力。

</SiteSection>

<SiteSection title="典型使用方式" description="Agents 并不是孤立存在，而是与 Notebook、Files 和治理链路共同工作。">

在典型工作流里，管理员会先创建并接入智能体，随后团队成员在任务中选择合适的执行能力，而运行过程和结果又会继续回流到任务界面、文件资产和审计链路中。也正因为如此，Agents 从来都不是孤立模块，而是整条运行链路的关键节点。

</SiteSection>
