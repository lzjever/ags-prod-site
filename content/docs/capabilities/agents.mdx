# Agents

Agents 模块负责让外部或内部智能体以标准协议接入项目。

<SiteSection title="产品定位" description="它不是资源清单页，而是项目级执行能力接入面。">

1. 支持 external agent 与 internal agent 的接入与管理。
2. 通过统一协议把执行请求、trace 事件与 artifact 回传串起来。
3. 成为 Chat 与 Notebook 调用的能力底座。

</SiteSection>

<SiteSection title="为什么重要" description="这一步把个人脚本式 agent 转化为团队可用的正式能力。">

1. 用户不再依赖个人命令行环境。
2. Agent 可以在项目结构中被发现、调用和治理。
3. 为托管执行与 sandbox 演进保留了清晰骨架。

</SiteSection>

<SiteSection title="典型使用方式" description="Agents 并不是孤立存在，而是与 Notebook、Files 和治理链路共同工作。">

1. 管理员先创建并接入智能体。
2. 团队成员在任务中选择合适的智能体执行。
3. 运行过程与结果继续回流到任务界面、文件资产和审计能力中。

</SiteSection>
