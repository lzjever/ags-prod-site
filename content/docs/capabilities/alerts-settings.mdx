# Alerts 与 Settings

Alerts 与 Settings 分别承担治理提醒与项目配置维护职责。

<SiteSection title="Alerts" description="Alerts 是治理信号，不是发布流水线或告警编排系统。">

Alerts 承接的是与治理相关的提醒信号，而不是一个复杂的告警编排中心。它的价值在于，当项目里出现值得关注的异常热点或治理提示时，团队能够更快把注意力拉回到真正需要处理的问题上；而在没有异常时，它又不会用无意义的信息制造噪音。

</SiteSection>

<SiteSection title="Settings" description="Settings 用于维护项目基础信息和治理边界，是项目生命周期管理的重要组成部分。">

Settings 负责维护项目基础配置和治理边界，它的意义不是再做一个复杂后台，而是为项目生命周期提供稳定、清晰的管理入口。对企业产品来说，这种克制反而很重要，因为它让设置真正服务于协作和治理，而不是把用户带进新的复杂度里。

</SiteSection>
