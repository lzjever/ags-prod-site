# Audit

Audit 面向管理员回答四个问题：

1. 最近发生了什么变化？
2. 谁改了什么？
3. 哪些资源有异常？
4. 接下来要去哪里处理？

<SiteSection title="当前角色" description="Audit 已经成为当前 MVP 中承接运行时可见内容与治理证据的主要收口点。">

1. 审计主表与详情抽屉。
2. 基础筛选。
3. 配置变更、异常事件与关键治理记录。

</SiteSection>

<SiteSection title="为什么重要" description="没有 Audit，治理就只能停留在配置层而不能形成证据链。">

1. 帮助管理员完成项目级排查与审查。
2. 作为 Usage、Policy 与执行结果的证据补充层。

</SiteSection>
