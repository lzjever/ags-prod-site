# Audit

Audit 面向管理员回答四个问题：

1. 最近发生了什么变化？
2. 谁改了什么？
3. 哪些资源有异常？
4. 接下来要去哪里处理？

<SiteSection title="核心作用" description="Audit 用于把项目中的关键变更、异常与治理动作集中展示给管理员。">

1. 审计主表与详情抽屉。
2. 基础筛选。
3. 配置变更、异常事件与关键治理记录。

</SiteSection>

<SiteSection title="为什么重要" description="没有 Audit，管理员就很难完整回看变化过程并进行排查。">

1. 帮助管理员完成项目级排查与审查。
2. 帮助把 Usage、Policy 与执行结果串联起来理解。

</SiteSection>

<SiteSection title="典型使用场景" description="Audit 最适合在资源限制调整、异常排查和责任归属确认时使用。">

1. 查看某次策略变更是谁发起、作用于哪个资源。
2. 根据异常事件继续跳转到成员、策略或端点页面。
3. 结合 Usage 与任务结果判断是否需要进一步治理动作。

</SiteSection>
