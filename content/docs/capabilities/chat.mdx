# Chat

Chat 是项目内的对话式 AI 入口，负责解决“先把问题问出来、先把讨论开始起来”这件事。

<SiteSection title="使用价值" description="Chat 让团队可以在项目中更自然地开始日常 AI 工作。">

Chat 的作用不只是让用户更方便地提问，而是让 AI 更自然地进入团队日常工作。成员可以先在当前项目上下文中快速开始，再继续进入任务页面、文件页面和资源管理页面。

</SiteSection>

<SiteSection title="它适合什么样的工作" description="Chat 更适合高频、轻量、需要快速反馈的场景。">

当用户需要快速问答、澄清信息、整理思路、连续讨论某个问题，或基于当前项目上下文反复迭代时，Chat 往往是最自然的入口。它的优势不在于承接最长的执行链，而在于让成员不必先组织复杂任务，就能先把工作启动起来。

</SiteSection>

<SiteSection title="产品定位" description="它应该是用户进入项目后最自然的 AI 入口。">

Chat 应该是用户进入项目后最自然的 AI 入口。它承接的是快速分析、问答和日常迭代这类高频需求，让用户能够基于 Endpoint 或 External Agent 直接发起对话，并优先得到结果，而不是先被复杂任务设置挡在门外。

</SiteSection>

<SiteSection title="为什么重要" description="Chat 不是脱离项目规则单独存在的一页。">

Chat 的作用不只在于低门槛，还在于它没有脱离项目规则。每次发送前都会绑定可用执行对象，对附件、多模态能力和错误映射保持前置判断，并与 Notebook、API 路径共享同一套资源限制逻辑。因此它看起来轻，却始终建立在同一个项目规则之上。

</SiteSection>

<SiteSection title="它和 Notebook 的分工" description="一个适合快速开始，一个适合持续推进。">

Chat 和 Notebook 不是互相替代的关系。Chat 更适合在项目中快速开始工作、反复交流和即时得到反馈；Notebook 更适合把更复杂的任务、输入资料、执行过程和结果整理成可回看的任务记录。对很多团队来说，最自然的路径往往是先在 Chat 中澄清问题，再把需要继续推进的工作带入 Notebook。

</SiteSection>
