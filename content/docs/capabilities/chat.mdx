# Chat

Chat 是项目内对话式 AI 使用面，负责解决“快速开始”和“低门槛使用”的问题。

<SiteSection title="使用价值" description="Chat 让团队可以在项目中更自然地开始日常 AI 工作。">

Chat 的价值不只是让用户更方便地提问，而是让 AI 能更自然地进入团队日常工作。成员可以先在当前项目上下文中快速开始，再继续进入更完整的任务执行和治理链路。

</SiteSection>

<SiteSection title="产品定位" description="它应该是用户进入项目后最自然的 AI 入口。">

Chat 应该是用户进入项目后最自然的 AI 入口。它承接的是快速分析、问答和日常迭代这类高频需求，让用户能够基于 endpoint 或 external agent 直接发起对话，并优先获得结果，而不是先被复杂执行协议挡在门外。

</SiteSection>

<SiteSection title="为什么重要" description="Chat 不是独立治理体系，而是 endpoint 统一约束链路的一部分。">

Chat 的价值不只在于低门槛，还在于它没有脱离项目治理体系。每次发送前都会绑定可用执行对象，对附件、多模态能力和错误映射保持前置判断，并与 Notebook、API 路径共享同一套治理逻辑。因此它看起来轻，却始终建立在稳定的项目结构之上。

</SiteSection>
