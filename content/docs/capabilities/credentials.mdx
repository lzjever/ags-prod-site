# Credentials

Credentials 模块用于集中管理对接第三方模型和服务所需的项目级凭据。它解决的不是“把密钥存在哪儿”这么简单的问题，而是让模型接入、智能体接入和后续轮换动作都有同一个可信入口，而不是继续散落在个人环境、浏览器记忆或脚本配置里。

<SiteSection title="它解决什么问题" description="没有统一凭据层，资源接入和后续维护都会变得非常脆弱。">

Credentials 把原本容易散落在个人环境和配置文件里的连接信息，收拢成项目级对象。这样一来，Endpoint 可以明确引用哪一份凭据，智能体接入也可以围绕同一个连接基础继续配置。对团队来说，更重要的是：当密钥需要轮换、替换或停用时，后续影响范围不再只能靠人工回忆，而是可以顺着同一个对象继续查。

</SiteSection>

<SiteSection title="你通常会在这里管理什么" description="这里管理的不是抽象凭据，而是项目真正要用到的连接基础。">

管理员通常会在这里维护模型服务的 API key、第三方平台访问令牌，或者某些服务接入所需的认证信息。页面除了展示名称和类型，还会让团队知道这份凭据最近什么时候轮换过、目前被哪些资源引用。这样做的意义，不只是把密钥录进去，而是让每一份连接信息都可以被长期维护。

</SiteSection>

<SiteSection title="它怎样进入后续工作" description="Credentials 本身不是终点，而是后面很多配置页的基础。">

在实际使用中，管理员会先录入项目级凭据，再在 Endpoint 创建时完成绑定。后续 Chat、Notebook 和 Agents 并不会直接关心密钥字符串本身，它们关心的是引用了哪一个已经准备好的资源入口。也正因为如此，Credentials 更像接入底座：它不直接生成业务结果，但它决定了后面的模型调用和智能体执行能不能稳定地继续下去。

</SiteSection>
