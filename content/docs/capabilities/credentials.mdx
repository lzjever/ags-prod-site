# Credentials

Credentials 模块用于集中管理对接第三方模型和服务所需的项目级凭据。

<SiteSection title="产品意义" description="它帮助把散落的连接配置变成项目级治理对象。">

Credentials 把原本容易散落在个人环境和配置文件里的连接信息，收拢成项目级治理对象。凭据因此可以与 endpoints 和 agents 保持明确关联，也能显著降低安全与维护成本，让接入能力拥有更稳定的基础。

</SiteSection>

<SiteSection title="当前阶段" description="当前 secret handling 已覆盖主要交互与权限门禁，但仍会继续演进。">

当前阶段，Credentials 已经覆盖了主要交互和权限门禁，能够支持项目中的核心接入场景。与此同时，更强的 secret 安全姿态仍然是后续持续演进的方向，因此这部分能力已经具备可用基础，但不会停留在现状。

</SiteSection>

<SiteSection title="典型使用方式" description="Credentials 本身不是终点，而是支撑模型接入和智能体执行的基础能力。">

在实际使用中，管理员会先录入项目级凭据，然后在 Endpoint 创建时完成绑定，后续 Chat、Notebook 和 Agents 则在统一治理条件下继续复用这些接入能力。也正因为如此，Credentials 更像一层基础设施，而不是孤立功能。

</SiteSection>
