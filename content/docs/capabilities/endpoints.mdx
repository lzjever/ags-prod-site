# Endpoints

Endpoints 是当前项目级治理主线的核心资源对象。

<SiteSection
  title="需求定位"
  description="后续 Chat、Notebook 与 API 路径都围绕同一套 endpoint 资源工作。"
>

Endpoints 负责统一管理项目可用的模型资源，把能力边界、资源来源和成本边界集中配置起来。这样一来，后续的 Chat、Notebook 与 API 路径都能围绕同一套资源工作，而 Usage、Policy、Audit 也就有了共同的主对象。

</SiteSection>

<SiteSection title="关键特征" description="Endpoint 是治理主线中的核心资源对象，而不只是普通配置列表。">

Endpoint 不是普通配置列表，而是治理主线里的核心资源对象。它既承接资源的创建、维护和导入导出，也通过 capability-first 的 provider / model 选择方式把接入过程组织得更清晰，同时让 Chat 只暴露真正可用于对话的资源能力。这样的设计，会让整个产品的资源表达更加统一。

</SiteSection>

<SiteSection title="为什么 Endpoint 会影响整条主线" description="资源一旦被定义成正式对象，后续使用和治理才能真正对应起来。">

当 Chat、Notebook 和 Agents 都围绕同一组 Endpoint 工作时，系统才能把“使用了什么资源”“是否应该被限制”“消耗如何累积”“异常发生在什么对象上”这些问题继续接起来。也正因为如此，Endpoint 页面对项目的意义并不只是配置模型，而是建立整条资源主线的起点。

</SiteSection>
