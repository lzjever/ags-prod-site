# Files

Files 不只是附件区，而是 AgentSmith 的项目资产层。它负责把资料、任务输入和执行结果持续组织在同一项目结构中，让团队既能在浏览器中工作，也能与本地目录保持协作关系。

<SiteSection title="它解决什么问题" description="没有稳定资产层，智能体输入和结果就很难被团队真正复用。">

很多 AI 工作之所以难以长期延续，不是因为当下跑不出结果，而是因为输入资料和输出结果没有稳定落点。文件散落在个人电脑、临时目录和一次性附件里时，团队很难知道哪些资料是正式上下文，哪些结果可以继续复用。Files 解决的，就是把这些信息持续保留在项目中。

</SiteSection>

<SiteSection title="为什么它不仅仅是文件管理" description="它是人类成员与智能体任务共同依赖的上下文层。">

项目成员会在这里上传和整理资料，Notebook 任务也会从这里读取输入、把结果回写进来。本地挂载能力又让浏览器中的文件库和本地工作目录保持协作。这样一来，Files 既服务人的工作，也服务智能体的工作，真正把资料层变成项目工作流的一部分。

</SiteSection>

<SiteSection title="它如何进入长期任务链路" description="Files 会持续连接输入、执行和结果沉淀。">

一项任务可以从 Files 中读取资料作为输入，运行后再把新的产物写回项目文件库，后续任务又能继续复用这些结果。对企业团队来说，这意味着 AI 结果不会只停留在一次交互中，而会逐步变成可以积累、可以协作、也可以继续加工的项目资产。

</SiteSection>
