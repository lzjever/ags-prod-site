# Files

Files 不只是附件区，而是 AgentSmith 的项目文件区。它负责把资料、任务输入和执行结果整理到同一个项目里，让团队既能在浏览器中工作，也能和本地目录保持衔接。

<SiteSection title="它解决什么问题" description="没有稳定的文件区，输入资料和结果就很难被团队真正复用。">

很多 AI 工作之所以难以继续下去，不是因为当下跑不出结果，而是因为输入资料和输出结果没有稳定落点。文件散落在个人电脑、临时目录和一次性附件里时，团队很难知道哪些资料应该继续保留，哪些结果可以交给下一个成员继续用。Files 解决的，就是把这些信息稳定地保留在项目中。

</SiteSection>

<SiteSection title="为什么它不仅仅是文件管理" description="它既服务成员，也服务任务执行。">

项目成员会在这里上传和整理资料，Notebook 任务也会从这里读取输入、把结果回写进来。本地挂载能力又让浏览器中的文件库和本地工作目录保持衔接。这样一来，Files 不只是“存文件的地方”，而是团队整理资料、任务读取资料、结果回写资料的共同入口。

</SiteSection>

<SiteSection title="它如何进入任务流程" description="Files 会把输入资料、执行过程和结果文件接在一起。">

一项任务可以从 Files 中读取资料作为输入，运行后再把新的产物写回项目文件库，后续任务又能继续复用这些结果。对团队来说，这意味着 AI 结果不会只停留在一次交互中，而会变成可以积累、可以协作、也可以继续加工的项目文件。

</SiteSection>
