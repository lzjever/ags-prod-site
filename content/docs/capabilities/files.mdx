# Files

Files 不只是附件区，而是 AgentSmith 的项目上下文资产层。

<SiteSection title="它解决什么问题" description="智能体输入与产出如果只停留在临时目录里，就很难沉淀、复用与协作。">

Files 解决的是智能体输入与产出难以沉淀的问题。如果所有资料都停留在临时目录和个人电脑里，团队就很难真正复用和协作。通过项目文件库、目录结构和本地挂载能力，Files 把 Web 工作流与本地工作流接了起来，也为 Notebook 与 Artifacts 提供了持久化落点。

</SiteSection>

<SiteSection title="为什么重要" description="它让通用智能体运行结果逐步从一次性临时文件变成项目资产。">

输入文件不会继续散落在临时目录中，一次任务产生的结果也可以直接成为下一次任务的输入，本地工作流与平台工作流也能够持续衔接。对企业团队来说，AI 结果不再只是“看过就结束”，而是能够长期积累的项目资产。

</SiteSection>

<SiteSection title="典型使用方式" description="Files 同时连接人类成员的文件操作和智能体任务的输入输出。">

在典型使用方式里，项目成员会先创建文件库并上传基础资料，智能体任务再读取这些文件、生成产物并回写文件库，而后续任务又会继续复用已有输入和产出。这样一来，Files 同时连接了人的工作和智能体的工作，让项目资产形成持续积累。

</SiteSection>
