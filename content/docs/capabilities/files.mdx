# Files

Files 不只是附件区，而是 AgentSmith 的项目上下文资产层。

<SiteSection title="它解决什么问题" description="智能体输入与产出如果只停留在临时目录里，就很难沉淀、复用与协作。">

1. 管理项目文件库与目录结构。
2. 支持本地挂载说明，连接 Web 与本地工作流。
3. 承接 Notebook 与 Artifacts 的持久化落点。

</SiteSection>

<SiteSection title="为什么重要" description="它让通用智能体运行结果逐步从一次性临时文件变成项目资产。">

1. 输入文件不再散落。
2. 结果文件可以直接成为下一次任务输入。
3. 本地工作流与平台工作流能自然衔接。

</SiteSection>

<SiteSection title="典型使用方式" description="Files 同时连接人类成员的文件操作和智能体任务的输入输出。">

1. 项目成员创建文件库并上传基础资料。
2. 智能体任务读取文件、生成产物并回写文件库。
3. 后续任务继续复用已有文件和产物，形成持续积累。

</SiteSection>
