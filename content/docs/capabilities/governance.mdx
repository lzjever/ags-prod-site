# Governance 总览

AgentSmith 当前唯一治理主线是：`项目级 LLM endpoint 统一约束链路`。

<SiteSection
  title="主线对象"
  description="治理不是围绕每个界面各自展开，而是围绕项目级 endpoint 资源展开。"
>

1. Endpoints 负责资源配置入口。
2. Resource Policy 负责访问与限制。
3. Usage 负责用户视角的用量理解。
4. Audit 负责管理员视角的事件回看与追溯。

</SiteSection>

<SiteSection
  title="为什么这条主线重要"
  description="如果 Chat、Notebook、API 各自长出一套治理系统，产品就会很快失控。"
>

1. 治理语言更统一。
2. 审查与追溯更稳定。
3. 后续扩展更清晰。

</SiteSection>
