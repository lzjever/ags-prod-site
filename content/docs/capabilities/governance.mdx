# Governance 总览

AgentSmith 当前唯一治理主线是：`项目级 LLM endpoint 统一约束链路`。

<SiteSection title="治理结构" description="治理能力会围绕资源定义、限制执行、用量观察和审计回看持续展开。">

AgentSmith 的治理能力不是若干分散设置的集合，而是围绕同一组资源对象持续展开。资源如何定义、限制如何生效、消耗如何被观察、变化如何被回看，都会在同一项目中保持连续。

</SiteSection>

<SiteSection
  title="主线对象"
  description="治理不是围绕每个界面各自展开，而是围绕项目级 endpoint 资源展开。"
>

AgentSmith 的治理主线不是围绕每个界面各自长出一套规则，而是围绕项目级 endpoint 资源统一展开。Endpoints 负责资源配置入口，Resource Policy 负责访问与限制，Usage 承接用户视角的用量理解，Audit 则负责管理员视角的事件回看与追溯。正因为它们围绕同一个对象工作，整条治理链路才足够稳定。

</SiteSection>

<SiteSection
  title="统一治理主线"
  description="Chat、Notebook、API 与资源管理会围绕同一条治理链路持续工作。"
>

Chat、Notebook 和 API 不会各自形成独立治理体系。资源定义、访问限制、用量观察和审计回看会沿同一条主线持续工作，因此项目中的资源状态和限制结果会更容易保持一致。

</SiteSection>

<SiteSection title="治理如何落到实际使用中" description="治理主线最终要在项目成员的真实使用过程中持续生效。">

真正有价值的治理，不是只停留在配置页面里。Endpoint 决定哪些模型资源可被使用，Resource Policy 决定谁可以如何使用这些资源，Usage 帮助成员理解消耗，Audit 帮助管理员回看变化与异常。也就是说，治理会持续出现在真实使用过程里，而不是只在出问题时才被想起。

</SiteSection>
