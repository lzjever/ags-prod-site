# Members

Members 是项目权限控制与成员治理的核心界面。

<SiteSection title="产品定位" description="它用于管理成员、处理加入申请，并清晰区分 owner / admin / member 边界。">

Members 是项目权限控制与成员治理的主链路。它承接成员加入申请、project admin 的日常管理，以及权限来源和有效访问范围的核对，让协作边界真正落到具体成员身上，而不是停留在抽象角色名称里。

</SiteSection>

<SiteSection title="为什么重要" description="成员治理既不能过度中心化，也不能失去最终责任边界。">

成员治理既不能过度中心化，也不能失去最终责任边界。owner 需要保留最终治理责任，admin 承接日常管理，而所有实际门禁又必须以 permission 结果为准，而不是停留在角色文案本身。只有这样，权限边界才会既清晰又可执行。

</SiteSection>
