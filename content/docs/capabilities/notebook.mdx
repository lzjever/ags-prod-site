# Notebook

Notebook 不是“再做一个 Chat”，而是 AgentSmith 当前最重要的统一智能体操作界面。

<SiteSection
  title="核心对象"
  description="Notebook 的用户心智应始终围绕 Task、Inputs、Conversation、Trace 与 Artifacts。"
>

1. 用户以任务视角，而不是命令行视角操作智能体。
2. 输入来源可以来自 Files、URL 和历史 artifacts。
3. 运行过程可以通过 trace 回看，结果可以沉淀为后续可复用资产。

</SiteSection>

<SiteSection
  title="为什么是统一智能体入口"
  description="它把原本分散在目录、脚本、终端输出与临时文件里的过程组织进同一个项目内工作流。"
>

1. 创建 Task。
2. 添加上下文输入。
3. 绑定 Agent 与 Endpoint。
4. 观察执行过程与 trace。
5. 保存或复用产物。

</SiteSection>
