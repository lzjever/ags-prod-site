# Notebook

Notebook 不是“再做一个 Chat”，而是 AgentSmith 当前最重要的统一智能体操作界面。

<TalkTrack
  audience="Value"
  title="为什么这项能力值得单独讲"
  talk="Notebook 的真正价值，在于它把复杂执行从临时过程升级成长期任务。团队不只是得到一次结果，而是得到一条可以继续回看、继续协作、继续复用的执行链路。"
  proof="这也是 AgentSmith 与普通 AI 入口最容易拉开差异的地方: 它不只生成结果，也组织结果是如何产生和如何继续被使用的。"
/>

<SiteSection
  title="核心对象"
  description="Notebook 的用户心智应始终围绕 Task、Inputs、Conversation、Trace 与 Artifacts。"
>

Notebook 的核心，不是再做一个聊天窗口，而是让用户以任务视角而不是命令行视角来操作智能体。输入可以来自 Files、URL 和历史 artifacts，运行过程可以通过 trace 回看，结果又会继续沉淀为后续可复用资产。这样一来，复杂执行就真正进入了项目工作流。

</SiteSection>

<SiteSection
  title="为什么是统一智能体入口"
  description="它把原本分散在目录、脚本、终端输出与临时文件里的过程组织进同一个项目内工作流。"
>

Notebook 之所以能成为统一智能体入口，是因为它把原本分散在目录、脚本、终端输出与临时文件里的过程，组织进了同一个项目内工作流。用户会围绕 Task 创建、上下文输入、Agent 与 Endpoint 绑定、执行观察以及产物复用这些动作持续工作，而不是每次都重新拼装一条执行链路。

</SiteSection>
