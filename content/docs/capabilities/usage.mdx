# Usage

Usage 的目标很克制：回答用户“我在用哪些资源、用了多少、离上限还有多远”。

<SiteSection title="产品边界" description="Usage 面向普通成员，不承载管理员排障与复杂诊断。">

Usage 面向的是普通成员，而不是承担管理员排障任务。它需要克制地展示个人视角下的使用量、limit summary、趋势变化与资源切换，让用户能快速理解自己当前的资源状态，而不会被复杂诊断信息打断。

</SiteSection>

<SiteSection title="为什么重要" description="它帮助用户在被拒绝前就调整行为，而不是等错误发生后才知道超限。">

它的重要性在于，让用户在真正被拒绝之前就能感知变化并调整行为。无论是空数据、接近上限还是已经命中限制，Usage 都应该用低心智的方式把状态表达清楚，而不是把管理员视角的诊断信息强塞给普通用户。

</SiteSection>
