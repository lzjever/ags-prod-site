# 智能体连接信息

<SiteSection title="你会在这里做什么" description="这里用于复制连接地址与服务密钥，并在本地 runner 中完成接入。">

这一步更像是把一项原本只存在于本地环境里的执行能力，正式接入到项目体系中。用户会在这里看到连接地址、服务密钥和必要的接入信息，然后把这些内容带到本地 runner 完成验证。整个过程是明确、受控的，而不是依赖个人口口相传的临时做法。

</SiteSection>

<SiteSection title="关键看点" description="连接信息对话框最适合说明 external agent 与本地 runner 之间如何建立正式连接。">

这张对话框最有价值的地方，在于它把连接过程做成了清晰可管理的动作。连接地址和服务密钥被集中展示，意味着外部执行能力并不是“接上就算了”，而是被明确纳入项目级智能体体系，方便后续使用、治理和排查。

</SiteSection>
