# 创建智能体对话框

<SiteSection title="这个动作的意义" description="创建智能体意味着把执行能力正式接入项目。">

很多团队已经拥有自己的 runner、脚本或外部执行程序，但如果这些能力没有被正式接入系统，它们就很难成为团队共享的工作能力。创建智能体对话框承担的，就是把这种执行能力先定义成项目对象，再让后续任务和对话能够持续调用它。

</SiteSection>

<SiteSection title="你需要提供什么信息" description="名称、模式和交互方式会决定团队如何理解和使用这个智能体。">

用户通常需要在这里定义智能体名称、运行模式、基础说明和交互方式。名称决定团队如何识别它，运行模式决定它属于外部接入还是平台托管方向，交互方式则决定后续成员在使用时会期待怎样的行为。

</SiteSection>

<SiteSection title="创建后会发生什么" description="智能体建立之后，还会继续进入连接、任务执行和治理链路。">

智能体对象建立完成后，团队通常还会继续查看连接信息、补充服务密钥或进入具体任务中选择它来执行工作。后续运行状态、trace、产物和相关审计记录，也都会围绕这个对象继续发生。因此，创建智能体对话框定义的并不是一个单独配置项，而是一项后续会长期参与项目工作的能力。

</SiteSection>
