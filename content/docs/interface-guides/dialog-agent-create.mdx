<SiteSection title="你会在这里做什么" description="这里用于创建新的智能体对象，并为后续连接信息与任务执行做准备。">

创建智能体不是简单填一张表单，而是在项目里正式定义一种可被团队调用的执行能力。名称、模式、说明和交互方式都在这里被明确下来，这意味着后续的连接、调用和治理都会围绕这个对象展开，而不是依赖个人脚本习惯。

</SiteSection>

<SiteSection title="关键看点" description="创建智能体对话框最适合说明项目如何把智能体能力纳入正式管理。">

这张页面很好地体现了 AgentSmith 的产品节奏: 先把能力接入清楚，再让团队稳定使用。对企业团队来说，这种顺序很重要，因为它让智能体从“有人会用的个人工具”变成“团队都能发现和调用的正式能力”。

</SiteSection>
