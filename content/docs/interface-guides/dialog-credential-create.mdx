# 创建凭据对话框

<StatusBadge state="已实现" />

<ScreenshotFrame
  preset="dialog-credential-create-guide"
  caption="创建凭据对话框用于录入模型服务或第三方平台所需密钥，是接入 Endpoint 之前的重要准备。"
/>

<SiteSection title="这个动作的意义" description="创建凭据意味着把敏感接入信息从个人环境移入项目治理范围。">

模型服务和第三方平台往往依赖密钥、令牌或其他敏感连接信息。如果这些信息继续散落在个人电脑或临时环境里，团队很难长期维护。创建凭据对话框承担的，就是把这些连接信息正式纳入项目管理范围。

</SiteSection>

<SiteSection title="你需要提供什么信息" description="凭据名称、类型和密钥内容会决定后续可以接入什么资源。">

用户通常会在这里录入凭据名称、类型以及实际密钥信息。这一步的意义，不只是把一段字符串保存起来，而是为后续 Endpoint 或其他外部服务接入准备可复用的基础对象。

</SiteSection>

<SiteSection title="创建后会发生什么" description="凭据建立后，后续资源接入与轮换维护都会围绕它继续发生。">

凭据创建完成后，可以在凭据管理页继续维护，并被 Endpoint 或其他连接配置持续引用。后续如果需要轮换、替换或回看相关变更，也都能沿着同一对象继续处理。这让敏感连接信息终于进入了长期可维护的项目链路。

</SiteSection>
