<SiteSection title="你会在这里做什么" description="这里用于新增可复用的项目凭据，并为后续模型接入做准备。">

这一步承接的是项目最敏感、也最容易失控的一类信息: 凭据。用户会在这里完成录入，然后回到凭据管理页持续维护，再在 Endpoint 或其他接入配置中复用它。这样做的意义不是“把密钥存进系统里”，而是让敏感连接信息进入统一治理范围。

</SiteSection>

<SiteSection title="关键看点" description="创建凭据对话框最适合说明敏感接入信息如何被纳入统一项目管理。">

这张对话框的重点不在于展示原始密钥，而在于强调受控录入和后续复用。对团队来说，这会让模型接入、外部能力连接和后续运维都有统一基础，也减少凭据散落在个人环境里的风险。

</SiteSection>
