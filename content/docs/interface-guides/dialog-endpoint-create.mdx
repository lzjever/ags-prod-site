# 创建 Endpoint 对话框

<StatusBadge state="已实现" />

<ScreenshotFrame
  preset="dialog-endpoint-create-guide"
  caption="创建 Endpoint 对话框用于定义模型接入方式、协议与凭据绑定，是接入模型资源的重要步骤。"

<SiteSection title="这个动作的意义" description="创建 Endpoint 意味着把模型资源正式纳入项目。">

项目里的对话、任务和智能体执行都需要清楚地知道应该调用哪一类模型资源。创建 Endpoint 对话框承担的，就是把这种资源先定义成正式对象，让后续使用、限制、用量和审计都围绕同一资源继续对应起来。

</SiteSection>

<SiteSection title="你需要提供什么信息" description="Provider、model 和 credential 会共同决定资源如何进入项目。">

用户通常需要在这里指定 provider、model、协议方式以及对应的 credential。无论采用引导式配置还是手动填写，这一步的本质都是在说明“项目接下来可以通过什么方式访问这一类模型能力”。

</SiteSection>

<SiteSection title="创建后会发生什么" description="一旦资源建立，后续调用、限制和回看都会围绕它展开。">

Endpoint 创建完成后，Chat、Notebook 和 Agents 都可以继续使用这项资源；资源策略会继续决定它的访问边界；Usage 和 Audit 也会继续记录围绕它发生的真实事实。换句话说，这个对话框建立的是整条资源主线的起点。

</SiteSection>
