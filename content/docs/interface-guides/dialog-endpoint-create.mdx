<SiteSection title="你会在这里做什么" description="这里用于新增模型接入方式，并把它纳入项目统一治理链路。">

用户会在这里正式定义一项新的模型接入方式，并决定它如何进入项目的统一资源体系。无论选择引导式配置还是手动方式，这个动作都会明确 provider、model、credential 等关键关系，为后续聊天、任务和智能体调用建立可靠基础。

</SiteSection>

<SiteSection title="关键看点" description="创建 Endpoint 对话框最适合说明模型资源接入是如何被正式纳入治理主线的。">

这张页面真正说明的是: 模型资源接入并不是一个零散配置，而是治理主线的开端。一旦 Endpoint 在这里被定义清楚，后续可用能力、访问策略、用量观察和审计回看都会围绕这个对象继续展开。

</SiteSection>
