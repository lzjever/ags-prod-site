<SiteSection title="这个动作的意义" description="创建文件库意味着为一类资料建立长期容器。">

项目里的资料和任务产物如果没有稳定容器，很容易继续散落在不同目录和不同成员的工作环境中。创建文件库对话框承担的，就是先为某一类资料建立清晰容器，让后续上传、挂载和结果沉淀拥有固定落点。

</SiteSection>

<SiteSection title="你需要提供什么信息" description="文件库的名称和定位，会影响后续成员如何理解和使用它。">

用户通常需要在这里定义文件库的基本信息，使团队后续能够理解这个文件库主要承接什么资料、适合与哪些任务或工作流一起使用。虽然输入项不多，但它决定了后续资料组织方式是否清晰。

</SiteSection>

<SiteSection title="创建后会发生什么" description="文件库建立后，上传、挂载和产物回写都会围绕它继续发生。">

一旦文件库建立完成，团队就可以继续上传文件、配置本地挂载，并让 Notebook 任务的产物回流到这里。这样一来，资料管理和结果沉淀就不再是临时动作，而会进入同一套项目资产结构中。

</SiteSection>
