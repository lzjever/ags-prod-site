<SiteSection title="你会在这里做什么" description="这里用于创建新的项目文件库，并开始沉淀长期可复用的文件资产。">

创建文件库意味着为一类资料建立正式容器，而不是临时找个地方上传文件。完成这一步之后，团队可以继续上传资料、执行本地挂载，也可以让后续任务产物自然回流到这里。文件资产从创建开始，就被放进可复用、可管理的项目结构中。

</SiteSection>

<SiteSection title="关键看点" description="创建文件库对话框最适合说明项目文件并不是临时上传区，而是长期资产容器。">

这张对话框很适合用来说明 Files 的价值: 它不是附件区，而是项目资产层。文件库一旦建立，后续上传、挂载和产物回写都会围绕这个容器展开，团队因此更容易把资料和结果长期沉淀下来。

</SiteSection>
