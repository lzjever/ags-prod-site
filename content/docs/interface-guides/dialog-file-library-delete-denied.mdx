# 文件库删除限制

<StatusBadge state="已实现" />

<ScreenshotFrame
  preset="dialog-file-library-delete-denied-guide"

<SiteSection title="这个对话框的作用" description="它负责在高风险删除动作前明确阻断错误操作。">

当一个文件库仍被引用、仍包含重要资料，或者删除动作会影响后续工作时，系统不能把这件事处理成普通确认。删除限制对话框承担的，就是在误删风险较高时明确阻断操作，并告诉用户为什么当前还不能删除。

</SiteSection>

<SiteSection title="你通常会在这里看到什么" description="删除被阻止的原因和需要满足的条件，是这个对话框最核心的信息。">

用户会在这里看到当前删除为什么会被拒绝、还存在哪些引用关系、以及系统要求满足哪些条件之后才能继续操作。它的重点不是再做一次确认，而是帮助用户先理解影响范围，避免对后续任务和资料造成破坏。

</SiteSection>

<SiteSection title="为什么它重要" description="文件资产一旦误删，影响的通常不是单个页面，而是整条后续工作链。">

文件库里保存的往往不是一次性附件，而是长期资料、任务输入和执行结果。删除限制对话框让系统在高风险情况下优先保护这些资产，避免某一次误操作直接切断后续任务、引用关系和项目沉淀。

</SiteSection>
