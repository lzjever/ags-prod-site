<SiteSection title="这个动作的意义" description="本地挂载信息对话框负责把浏览器中的文件库接到本地工作目录。">

很多团队在使用 AI 时，不会只停留在浏览器中处理文件。本地工具链、编辑器和脚本环境仍然是重要工作场景。挂载信息对话框承担的，就是把项目文件库与本地目录稳定接起来，让同一批资料能够在两种工作方式之间持续流动。

</SiteSection>

<SiteSection title="你会在这里看到什么" description="挂载所需的地址、路径和命令，是这个对话框最核心的内容。">

用户会在这里查看 filesystem 名称、metadata 地址以及多平台挂载命令，并据此在本地环境中完成挂载。对很多团队来说，这一步的意义在于：项目资料不再只能在浏览器里被看见，而可以进入更熟悉的本地工作流。

</SiteSection>

<SiteSection title="为什么它重要" description="它让文件库真正成为浏览器与本地环境之间的共同资产层。">

如果没有本地挂载能力，很多资料仍然只能以下载、上传和临时复制的方式来回移动。挂载信息对话框让这种协作方式变得更连续，也让项目文件库真正承担起“统一资产层”的角色。

</SiteSection>
