# 邀请成员对话框

<StatusBadge state="已实现" />

<SiteSection title="这个动作的意义" description="邀请成员意味着把新的协作者纳入当前项目边界。">

项目一旦进入多人协作阶段，每一位新成员的加入都不只是“多一个名字”，而是意味着新的访问边界、新的资源使用路径和新的审计主体。邀请成员对话框承担的，就是把这一步从一开始定义清楚。

</SiteSection>

<SiteSection title="你需要提供什么信息" description="成员身份与分组来源会决定后续的有效访问范围。">

用户通常会在这里录入成员身份信息，并指定其分组或角色来源。虽然界面动作很简单，但它会直接影响该成员后续能够进入哪些页面、使用哪些资源，以及在审计记录中以什么身份出现。

</SiteSection>

<SiteSection title="创建后会发生什么" description="成员加入后，后续权限、资源使用和审计记录都会围绕这一定义继续发生。">

一旦邀请完成，成员治理页、资源策略、实际使用权限和审计记录都会继续围绕这位成员展开。也正因为如此，这个对话框不是一项孤立的人事动作，而是项目协作边界进入系统的第一步。

</SiteSection>
