# 邀请成员对话框

<StatusBadge state="已实现" />

<ScreenshotFrame
  preset="dialog-member-invite-guide"

<SiteSection title="你会在这里做什么" description="这里用于发送成员邀请，并把成员带入当前项目的治理范围。">

1. 填写成员邮箱并选择成员分组。
2. 发送邀请后在成员列表中跟踪状态。
3. 配合成员治理页面继续调整访问范围。

</SiteSection>

<SiteSection title="关键看点" description="邀请成员对话框最适合说明项目成员不是随意加入，而是带着明确分组和边界进入。">

1. 邮箱、分组与角色来源在邀请阶段就被明确。
2. 页面体现了权限治理从成员进入项目时就开始生效。
3. 这也是解释成员治理为什么是项目级能力的重要入口。

</SiteSection>
