<SiteSection title="你会在这里做什么" description="这里用于创建长期任务，并选择执行智能体。">

这一步定义的不是一条一次性消息，而是一项可以持续回看和继续协作的任务。用户在创建时就会绑定执行智能体，也因此从一开始就为后续的 trace、artifacts 和结果沉淀打下结构基础。Notebook 的价值，正是从这里开始和普通对话入口拉开差异。

</SiteSection>

<SiteSection title="关键看点" description="创建任务对话框最适合用来说明 Notebook 与普通对话入口的区别。">

如果说 Chat 解决的是快速开始，那么 Notebook 从这一刻起解决的就是长期执行。创建任务时的每一个输入，都在说明这不是临时会话，而是一条能够被跟踪、被复用、也能持续沉淀结果的工作流。

</SiteSection>
