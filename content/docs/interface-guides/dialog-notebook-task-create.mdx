<SiteSection title="这个动作的意义" description="创建任务意味着把一次执行升级成可持续回看的工作对象。">

Notebook 中最重要的不是消息框，而是 Task。创建任务对话框承担的，就是让一次复杂执行从一开始就拥有明确对象，而不是作为一次临时调用被很快遗忘。任务一旦建立，后续输入、过程和结果都会围绕它持续保留。

</SiteSection>

<SiteSection title="你需要提供什么信息" description="任务标题和执行智能体会影响后续理解与运行方式。">

用户在这里通常需要定义任务标题，并选择要使用的执行智能体。标题决定了后续成员如何理解这项工作，智能体选择则决定了后续任务将由什么能力来执行。虽然输入项不多，但它们会直接影响这项任务后续的组织方式。

</SiteSection>

<SiteSection title="创建后会发生什么" description="任务建立之后，输入、trace 和 artifacts 都会继续围绕它展开。">

一旦任务创建完成，成员就可以继续补充输入资料、发起运行，并在任务详情页里查看状态、trace 和产物文件。也正因为如此，创建任务对话框并不是在发起一次普通操作，而是在开启一条可以继续推进、继续回看和继续复用的任务主线。

</SiteSection>
