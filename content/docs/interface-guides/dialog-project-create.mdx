<SiteSection title="你会在这里做什么" description="这里用于创建新项目，并设置基础规则。">

1. 设置项目名称与描述。
2. 作为后续治理与使用链路的起点。

</SiteSection>

<SiteSection title="关键看点" description="创建项目对话框最适合说明项目不是临时容器，而是正式的业务与治理边界。">

1. 名称、描述、可见性和加入策略在创建时就被明确下来。
2. 这意味着后续成员协作、资源治理和任务执行都会落到同一边界内。
3. 它是从 Workspace 进入 Project 的关键转换动作。

</SiteSection>
