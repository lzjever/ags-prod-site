<SiteSection title="你会在这里做什么" description="这里用于创建新项目，并设置基础规则。">

创建项目是从工作区进入具体业务上下文的关键一步。名称、描述和基础规则在这里被定义下来，随后成员协作、资源治理和任务执行都会围绕这个边界继续展开。也正因为如此，项目从一开始就不是临时容器，而是正式的工作与治理单位。

</SiteSection>

<SiteSection title="关键看点" description="创建项目对话框最适合说明项目不是临时容器，而是正式的业务与治理边界。">

这张页面看起来简单，但它实际上决定了后续所有事情的边界。项目一旦被创建，成员、资源、文件和任务都会落到同一个正式上下文中，这也是 AgentSmith 能把使用与治理放进同一套结构里的前提。

</SiteSection>
