# 创建项目对话框

<SiteSection title="这个动作的意义" description="创建项目意味着为后续所有工作建立共同边界。">

项目并不是简单的名称容器。它会成为后续成员协作、资源接入、任务执行、文件沉淀和审计回看的共同边界。创建项目对话框承担的，就是把这一边界从一开始定义清楚。

</SiteSection>

<SiteSection title="你需要提供什么信息" description="项目名称、说明和基础可见性，会直接影响后续理解和协作方式。">

用户在这里通常需要输入项目名称、项目说明以及基础属性。虽然这些信息看起来简单，但它们会决定成员如何识别这个项目、项目在工作区中如何被理解，以及后续协作和治理动作围绕什么边界继续发生。

</SiteSection>

<SiteSection title="创建后会发生什么" description="一旦项目建立，后续使用面和治理面都会围绕它展开。">

项目创建完成后，Chat、Notebook、Files、Agents、Endpoints、Members、Usage 和 Audit 等能力都会在这一边界内组织起来。换句话说，这个对话框决定的不是一个临时入口，而是整条项目工作流的起点。

</SiteSection>
