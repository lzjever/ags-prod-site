# 关键界面导览

这一组页面不是简单的截图索引，而是 AgentSmith 工作方式的直观入口。你可以从这里看到，一个项目如何把聊天、任务、文件、智能体和治理能力收拢到同一条业务主线中，也能更快理解团队为什么不需要在不同工具和流程之间来回切换。

如果你第一次接触 AgentSmith，建议先从日常使用最强的界面看起，再回到治理与配置页面。这样更容易先建立整体印象，再理解这套系统为什么既适合一线成员使用，也适合团队长期管理。

  description="如果你是第一次系统了解 AgentSmith，推荐按下面顺序阅读，而不是逐页随机跳转。"

先看 `项目总览`、`Chat 多轮会话` 和 `Notebook 任务详情`，你会很快明白 AgentSmith 不是把 AI 能力零散地塞进一个后台，而是让团队围绕项目上下文展开持续工作。接着再看 `文件库管理`、`智能体列表` 和相关创建页面，就能理解输入、执行和产物是如何被组织成长期可复用的协作链路。

最后再进入 `Endpoint 管理`、`资源策略`、`用量与限额`、`审计日志` 等页面，会更容易看懂治理为什么不是额外加上的后台功能，而是与日常使用天然连在一起的产品结构。

</SiteSection>
