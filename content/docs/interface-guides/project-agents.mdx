<SiteSection title="这个页面的作用" description="智能体列表页负责把执行能力清晰地摆到项目里。">

用户在这里可以直接看到当前项目已经接入了哪些智能体、它们是否在线、采用什么运行模式，以及各自适合承担什么样的工作。这使执行能力不再是隐藏在某个人环境里的外部依赖，而是正式进入项目视野的对象。对一个要长期使用智能体的团队来说，这页最重要的意义，是把“谁能执行、现在能不能执行”说清楚。

</SiteSection>

<SiteSection title="你通常会在这里看什么" description="状态、运行方式和后续入口，是这个页面最重要的信息。">

团队成员通常会先确认某个智能体是否可用，再判断它更适合进入哪一类任务。管理员则会继续进入创建、连接信息和配置相关页面，完成接入和维护。对第一次接触 AgentSmith 的用户来说，这一页也能帮助理解 External Agent 和 Internal Agent 的差别：前者接入的是已经跑在平台外面的执行端，后者代表的是由平台准备和回收执行环境的方向。

</SiteSection>

<SiteSection title="它和任务页面如何衔接" description="智能体列表页并不是终点，而是执行链路的起点之一。">

一旦智能体在项目中可用，成员就可以继续在 Chat 或 Notebook 中选择它来完成实际工作。任务运行时的过程、结果和后续记录，又会继续回流到任务历史、文件资产和审计链路中。因此，智能体列表页虽然本身是管理页面，但它直接关系到日常使用链路能否成立。很多团队在排查“任务为什么没跑起来”时，也会先回到这里确认智能体当前是否在线、是否接入正确。

</SiteSection>
