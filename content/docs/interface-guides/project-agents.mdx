# 智能体列表

<StatusBadge state="已实现" />

<SiteSection title="你会在这里做什么" description="这里用于查看智能体清单、判断当前可用状态，并继续进入连接信息或任务使用。">

智能体列表页把项目内可用的执行能力真正展示出来。团队成员可以在这里看到有哪些智能体可用、它们当前是否在线、适合用于聊天还是任务执行，以及接下来该去哪里完成连接或配置。相比“每个人自己记住一个脚本”或“把接入信息散落在文档里”，这种方式更像一个正式产品该有的能力入口。

它让智能体从个人环境中的隐性资源，变成团队可以发现、判断和调用的明确对象。

</SiteSection>

<SiteSection title="关键看点" description="这张页面最适合说明智能体能力如何被纳入项目级正式管理。">

这张页面最有说服力的地方，在于它把“接入智能体”和“使用智能体”之间的关系变得可见。团队不需要猜测某个能力是否可用，也不需要依赖口口相传的操作习惯，而是可以直接看到模式、状态和交互方式。这会显著降低协作门槛，也让执行能力真正进入项目管理范围。

</SiteSection>

<SiteSection title="关联模块" description="智能体列表连接接入、执行和治理三条主线。">

从这里继续往下走，用户会进入创建智能体、查看连接信息、配置服务密钥等接入动作；往另一边走，则会进入 Chat 和 Notebook 中的实际执行场景。与此同时，智能体的接入和使用又与 Endpoint、Credentials、Audit 等治理能力保持连续关系，这也是它在整站叙事里非常关键的一页。

</SiteSection>
