<SiteSection title="你会在这里做什么" description="这里用于回看关键变更、异常事件与资源操作。">

Audit 页面回答的是企业团队最常问的一类问题：最近发生了什么、是谁触发的、影响了哪些资源、接下来该去哪里处理。它把配置变化、异常事件和关键操作集中到同一视图里，让排查不再依赖零散日志和口头确认，而是能够在产品内完成基本的回看与追溯。

这也是 AgentSmith 治理能力最容易建立信任的一页，因为很多团队真正担心的并不是“能不能用”，而是出了问题之后能不能说清楚。

</SiteSection>

<SiteSection title="关键看点" description="审计页最适合说明 AgentSmith 如何让治理动作具备可回看、可排查的能力。">

这张页面最有说服力的地方，不只是把事件列出来，而是给了用户明确的排查抓手。筛选条件、事件列表和详情抽屉一起工作，让治理不再停留在“我设置过一条规则”，而是真正进入“我知道发生了什么，也知道下一步该怎么处理”的状态。

</SiteSection>

<SiteSection title="关联模块" description="Audit 是治理主线最终收口到管理员视角的重要界面。">

Audit 回看的对象来自 Endpoint、Policy、Members 等治理模块，而任务执行结果、异常波动和资源消耗也会在这里汇合。因此它并不是孤立的日志页，而是整条治理主线最终收口到管理员视角的地方。单条事件再通过详情抽屉展开，责任归属和上下文也就更容易被说清楚。

</SiteSection>
