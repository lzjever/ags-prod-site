<SiteSection title="这个页面的作用" description="审计日志页负责保留项目中的关键变化与异常事实。">

当项目中的资源配置、成员权限、限制命中或执行异常发生变化时，团队需要一个稳定的位置去回看这些事实。审计日志页承担的就是这项工作。它把关键事件集中保留在同一个项目上下文中，方便后续定位问题、解释变化和继续处理。对管理员来说，这一页最重要的不是“日志很多”，而是关键事件终于有了统一入口。

</SiteSection>

<SiteSection title="你通常会在这里看什么" description="事件列表、筛选条件和详情信息，是这个页面最重要的内容。">

管理员通常会在这里查看最近发生了什么、哪些资源受到影响、某次异常是由什么原因触发，以及后续是否需要继续进入资源、成员或任务相关页面。实际事件往往会覆盖策略更新、Endpoint 调用、成员状态变化、智能体相关动作和任务执行过程中的关键记录，因此这页提供的不是抽象日志，而是后续排查和追溯的事实依据。

</SiteSection>

<SiteSection title="为什么它重要" description="没有审计回看，治理很难形成闭环。">

企业级使用环境不仅要能配置资源和限制，还要能在需要时说清楚到底发生过什么。审计日志页的价值，就在于把这些事实持续保留下来，使资源治理、成员管理、任务执行和异常排查真正形成闭环，而不是只停留在当时的页面状态中。很多时候，真正让团队安心的不是“限制已经配置”，而是出了事之后知道去哪里把事情讲清楚。

</SiteSection>
