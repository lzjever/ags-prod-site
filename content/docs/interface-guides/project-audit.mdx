# 审计日志

<SiteSection title="界面概览" description="审计日志用于查看关键变更、异常事件和资源操作记录。">

Audit 页面集中展示配置变化、异常事件和关键操作记录。用户可以在这里查看最近发生了什么、是谁触发的、影响了哪些资源，以及下一步应从哪里继续处理。

对于需要持续管理项目变更的团队来说，这一页能够提供稳定的回看和追溯能力。

</SiteSection>

<SiteSection title="主要价值" description="审计日志帮助管理员快速定位事件、理解影响并继续处理。">

筛选条件、事件列表和详情抽屉共同构成了一套清晰的排查方式。用户不仅可以看到发生过哪些事件，也可以继续定位影响范围，并判断下一步该如何处理。

</SiteSection>

<SiteSection title="相关功能" description="审计日志与资源治理、成员管理和执行结果一起构成完整回看链路。">

Audit 会回看 Endpoint、资源策略、成员变更等管理活动，任务执行结果、异常波动和资源消耗也会在这里汇合。单条事件再通过详情抽屉展开后，责任归属和上下文也会更容易判断。

</SiteSection>
