# Chat 多轮会话

<SiteSection title="这个页面的作用" description="Chat 页面承接项目中最直接的 AI 交互。">

当用户需要快速提问、澄清信息、持续讨论或基于当前项目上下文发起多轮沟通时，通常会先进入 Chat。这里提供的是最轻量的工作入口，让成员不必先创建任务，也不必先组织复杂输入，就可以在项目范围内直接开始与 AI 交互。

</SiteSection>

<SiteSection title="你通常会在这里做什么" description="对话、切换线程和回看上下文，是这个页面最常见的动作。">

用户会在这里发起新对话、查看历史线程、切换不同会话，并围绕同一个项目上下文持续推进讨论。线程列表与当前会话内容并排展示，使用户能够更自然地保持上下文，也方便在多轮交流后快速回看之前已经得出的信息。

</SiteSection>

<SiteSection title="它和其他页面如何衔接" description="Chat 适合快速开始，复杂工作通常会继续进入 Notebook 和 Files。">

Chat 页面并不会脱离项目其他能力单独存在。它会继续使用项目中可用的模型资源或智能体，后续需要长期推进的工作通常会转向 Notebook，相关资料和结果也可以进一步进入 Files。对应的使用情况还会出现在用量和审计记录中，因此它既是快速入口，也是整条项目工作链的起点之一。

</SiteSection>
