# Chat 多轮会话

<SiteSection title="界面概览" description="Chat 页面用于发起对话、切换线程并查看回复结果。">

Chat 页面提供项目内最直接的交互方式。用户可以在当前项目上下文中发起多轮对话、切换线程并查看回复结果，而不需要额外离开当前工作环境。对于需要快速提问、澄清信息或持续讨论的场景，这一页通常是最常用的工作界面。

线程列表和当前会话内容并排展示，能够帮助用户在连续对话中保持上下文，也方便回看之前的讨论内容。

</SiteSection>

<SiteSection title="主要价值" description="Chat 提供了自然直接的交互方式，同时保持项目级边界。">

对话体验在这里保持自然，同时又始终处在项目范围内。用户看到的是简洁直接的交互方式，但对话所使用的资源、权限和后续记录都不会脱离当前项目。这样一来，团队既可以快速开始使用，也能保持统一管理。

</SiteSection>

<SiteSection title="相关功能" description="Chat 与项目中的资源、文件和任务能力保持连续关系。">

Chat 会调用项目中可用的 Endpoint 或智能体，输入和结果也可以进一步进入 Files 或 Notebook。对应的使用情况还会进入用量和审计记录，因此这并不是一块独立的对话区域，而是与整个项目工作链保持一致的一部分。

</SiteSection>
