<SiteSection title="你会在这里做什么" description="这里承接项目中的多轮对话、线程切换与结果查看。">

Chat 是团队最容易开始使用 AgentSmith 的入口。用户可以直接在项目上下文里发起多轮对话、切换线程、查看结果，而不必先理解复杂的执行协议或底层接入细节。对很多团队来说，这种低门槛起步非常重要，因为它决定了 AI 能不能真正进入日常工作，而不是停留在少数人的工具箱里。

这张页面所承接的，不只是“聊天功能”本身，更是一种更自然的产品承诺：让用户先获得结果，再逐步理解背后的能力体系。

</SiteSection>

<SiteSection title="关键看点" description="Chat 页最适合用来说明 AgentSmith 如何降低团队使用 AI 的门槛。">

真正有价值的地方在于，Chat 并没有为了“简单易用”而脱离项目体系。用户虽然看到的是轻量、自然的对话体验，但背后仍然沿用同一套项目资源、权限边界和治理规则。这样一来，团队既能享受快速上手的体验，也不会失去企业场景中最重要的可控性。

</SiteSection>

<SiteSection title="关联模块" description="Chat 看似轻量，但它背后连接的是完整的项目能力体系。">

一次对话背后调用的是 Endpoint 或 Agent，输入与结果又能继续流向 Files 和 Notebook，而实际使用情况还会进入 Usage 与 Audit 的观察范围。也因此，Chat 虽然是最轻的一层入口，却始终连接着 AgentSmith 的完整能力体系。

</SiteSection>
