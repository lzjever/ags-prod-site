<SiteSection title="你会在这里做什么" description="这里用于集中管理项目可用凭据及其轮换状态。">

1. 查看凭据名称、类型与轮换时间。
2. 新建、轮换与删除凭据。
3. 连接 endpoint 所需的 credential 引用。

</SiteSection>

<SiteSection title="关键看点" description="凭据页最适合说明平台如何把敏感连接信息纳入项目治理，而不是散落在个人环境里。">

1. 凭据以项目对象形式统一管理，便于轮换、查看和复用。
2. 页面本身不以暴露原始值为目标，而以受控使用和统一维护为目标。
3. 这为模型接入、智能体执行和权限治理提供了稳定基础。

</SiteSection>

<SiteSection title="关联模块" description="Credentials 自身不是终点，而是整个接入体系的基础层。">

1. Endpoint 创建和维护需要引用凭据。
2. Agents 与外部能力连接同样依赖受控凭据。
3. 凭据变更和使用行为继续进入 Audit 与治理范围。

</SiteSection>
