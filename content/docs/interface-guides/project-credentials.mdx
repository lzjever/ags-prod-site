<SiteSection title="你会在这里做什么" description="这里用于集中管理项目可用凭据及其轮换状态。">

凭据管理页把对接第三方模型和服务所需的敏感连接信息集中放进项目范围内统一维护。管理员可以在这里查看名称、类型和轮换状态，也可以持续完成新增、替换和清理，让接入能力不再散落在个人环境里。

</SiteSection>

<SiteSection title="关键看点" description="凭据页最适合说明平台如何把敏感连接信息纳入项目治理，而不是散落在个人环境里。">

这张页面想强调的并不是“看见密钥”，而是“受控地使用和维护密钥”。凭据一旦成为项目对象，就更容易被轮换、复用和审计，也为模型接入、智能体执行和权限治理提供了稳定基础。

</SiteSection>

<SiteSection title="关联模块" description="Credentials 自身不是终点，而是整个接入体系的基础层。">

Credentials 本身不是终点，它会继续被 Endpoint 和 Agents 引用，而凭据的变化和使用也会自然进入 Audit 与治理范围。也因此，它在产品里承担的是一层非常基础但非常关键的支撑角色。

</SiteSection>
