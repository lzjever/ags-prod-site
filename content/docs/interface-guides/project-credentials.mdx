<SiteSection title="这个页面的作用" description="凭据管理页负责把敏感连接信息纳入项目的正式管理范围。">

当项目需要接入外部模型服务或第三方平台时，团队不能把密钥和访问令牌分散保存在个人环境里。凭据管理页承担的，就是把这些敏感连接信息集中到项目范围内维护，使后续资源接入和轮换动作拥有稳定基础。

</SiteSection>

<SiteSection title="你通常会在这里做什么" description="查看、维护和轮换凭据，是这个页面最常见的动作。">

管理员通常会在这里查看项目当前有哪些可用凭据、它们分别对应什么服务、是否需要继续新增或替换，以及哪些接入对象正在引用这些凭据。这样做的意义，不只是把密钥录进去，而是确保后续模型资源和服务连接能够被持续维护。

</SiteSection>

<SiteSection title="为什么它重要" description="没有统一凭据层，资源接入很难长期稳定。">

凭据管理页让敏感连接信息能够以统一方式被维护和复用。Endpoint、智能体和其他外部服务配置都可以继续引用这里的凭据，而相关变更又会进入审计记录。这样一来，接入能力不再依赖个人记忆和临时保存方式，而会进入项目正式治理链路。

</SiteSection>
