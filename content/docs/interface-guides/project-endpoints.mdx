<SiteSection title="界面概览" description="Endpoint 页面用于配置和管理项目可用的模型资源。">

Endpoint 页面用于集中管理项目可用的模型资源。模型资源、供应商、能力边界和运行状态都会在这里被明确下来，因此后续无论是聊天、任务执行还是智能体调用，都可以基于同一组资源继续工作。

资源对象被集中管理之后，策略、用量和审计也就有了共同参照，后续管理会更加清晰。

</SiteSection>

<SiteSection title="主要价值" description="Endpoint 把模型资源集中到统一对象中，便于后续管理和使用。">

这里的重点在于把模型资源定义成清晰可管理的对象。Endpoint 建立之后，后续的限制、观察和追溯都可以围绕同一资源展开，团队在管理模型资源时也会更加稳定。

</SiteSection>

<SiteSection title="相关功能" description="Endpoint 与资源策略、用量、审计以及执行能力共同构成项目资源体系。">

Chat、Notebook 和 Agents 共用这里定义好的模型资源，Policy、Usage 和 Audit 也围绕这些资源持续展开。再加上凭据绑定、创建动作和后续治理页面，Endpoint 实际上把使用面和治理面牢牢接在了一起。

</SiteSection>
