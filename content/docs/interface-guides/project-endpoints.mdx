# Endpoint 管理

<StatusBadge state="已实现" />

<ScreenshotFrame
  preset="project-endpoints-guide"

<SiteSection title="你会在这里做什么" description="这里用于统一管理项目可用模型资源与相关配置。">

1. 管理项目可用模型资源。
2. 为后续 Policy、Usage、Audit 提供共同主对象。
3. 让治理表达更统一，避免多套平行治理体系。

</SiteSection>

<SiteSection title="关键看点" description="Endpoint 页最适合用来讲清楚 AgentSmith 的治理主线为什么围绕资源对象展开。">

1. 模型资源不是分散埋在聊天或任务页面里，而是在这里被集中管理。
2. 供应商、模型、能力边界与状态都围绕同一对象持续维护。
3. 这让后续的策略、用量、审计和执行使用都能指向同一主对象。

</SiteSection>

<SiteSection title="关联模块" description="Endpoint 是治理主线的起点，也是使用面的公共底座。">

1. Chat、Notebook 与 Agents 共享同一组可用模型资源。
2. Policy、Usage 与 Audit 围绕这些资源持续展开。
3. 创建 Endpoint、绑定凭据和后续治理动作共同构成完整链路。

</SiteSection>
