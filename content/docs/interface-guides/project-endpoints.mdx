<SiteSection title="你会在这里做什么" description="这里用于统一管理项目可用模型资源与相关配置。">

Endpoint 页面是整个治理主线的起点。模型资源、供应商、能力边界和运行状态都在这里被明确下来，因此后续无论是聊天、任务执行还是智能体调用，都能够指向同一组正式资源。对企业团队来说，这种统一入口非常重要，因为它避免了接入能力散落在不同页面和不同流程里。

一旦资源对象被集中管理，策略、用量和审计也就有了共同的参照系，整套治理表达会自然清晰很多。

</SiteSection>

<SiteSection title="关键看点" description="Endpoint 页最适合用来讲清楚 AgentSmith 的治理主线为什么围绕资源对象展开。">

很多平台的问题不在于不能接入模型，而在于接入之后缺少统一的管理中心。AgentSmith 在这里给出的答案很直接：把资源对象先定义清楚，再围绕它展开后续的限制、观察和追溯。这样一来，治理就不再像一堆零散规则，而更像一套有主线、有中心对象的产品结构。

</SiteSection>

<SiteSection title="关联模块" description="Endpoint 是治理主线的起点，也是使用面的公共底座。">

Chat、Notebook 和 Agents 共用这里定义好的模型资源，Policy、Usage 和 Audit 也围绕这些资源持续展开。再加上凭据绑定、创建动作和后续治理页面，Endpoint 实际上把使用面和治理面牢牢接在了一起。

</SiteSection>
