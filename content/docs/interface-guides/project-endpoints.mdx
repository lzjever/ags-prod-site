<SiteSection title="这个页面的作用" description="Endpoint 页面负责把模型资源定义成项目中的正式对象。">

团队在使用 Chat、Notebook 或智能体时，系统首先要知道可调用的模型资源是什么、来自哪里、处于什么状态。Endpoint 页面承担的就是这项工作。它让模型资源以统一对象进入项目，而不是零散地藏在每个功能页各自的设置里。

</SiteSection>

<SiteSection title="你通常会在这里做什么" description="配置资源、确认状态和理解后续影响，是这个页面最常见的动作。">

管理员通常会在这里创建新的模型资源、检查某个 Endpoint 当前是否可用，并理解它会被哪些页面和任务继续使用。因为 Endpoint 是项目资源，后续的限制、用量和审计也会围绕同一对象展开，所以这一页不仅仅是在“填一份模型配置”，而是在建立整条资源主线的起点。

</SiteSection>

<SiteSection title="为什么它重要" description="模型资源只有先成为正式对象，后续治理和复用才会稳定。">

如果模型配置散落在不同页面里，团队很难保证使用、限制和回看保持一致。Endpoint 页面的价值，就在于把模型资源先固定成项目对象，再让 Chat、Notebook、Agents、Resource Policy、Usage 和 Audit 围绕它继续工作。这样一来，资源定义、资源使用和资源治理才能真正连起来。

</SiteSection>
