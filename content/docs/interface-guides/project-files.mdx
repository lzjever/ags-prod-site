<SiteSection title="这个页面的作用" description="Files 页面负责集中管理项目中的资料与产物。">

Files 页面是项目的资产工作台。用户可以在这里查看文件库、浏览目录、检查文件详情，并完成上传、挂载和引用等操作。无论是原始资料、任务输入，还是任务生成的结果文件，都可以在这里持续组织和保存。

</SiteSection>

<SiteSection title="你通常会在这里做什么" description="整理资料、定位结果和连接本地目录，是这个页面最常见的动作。">

项目成员通常会在这里上传和整理基础资料，查找任务运行后生成的文件，确认某个文件是否已经进入项目资产层，并通过挂载能力让浏览器中的文件库与本地目录协同工作。对很多团队来说，这一页是把“浏览器中的 AI 工作”与“本地文件工作”真正接起来的关键位置。

</SiteSection>

<SiteSection title="为什么这个页面重要" description="它让输入资料和执行结果都拥有稳定落点。">

如果资料和结果只停留在临时目录或一次性附件里，团队很难真正复用它们。Files 页面的价值在于，它让任务输入和任务产物都进入同一套项目资产结构中。这样一来，今天产生的结果可以成为明天任务的输入，项目资料也能随着工作推进不断积累，而不是反复散失在个人环境里。

</SiteSection>
