<SiteSection title="你会在这里做什么" description="这里用于查看项目文件库、目录内容与本地挂载信息。">

Files 页面最重要的意义，是把“文件”从一次性附件升级成团队长期可复用的项目资产。用户在这里看到的不只是目录结构，而是一套能够承接输入资料、运行产物和后续协作的正式资产层。这样一来，项目里的知识和结果不再散落在个人电脑或临时目录里，而是能够被团队持续引用。

对很多企业场景来说，这一点非常关键，因为真正有价值的并不是某一次任务跑完，而是这些输入与输出能否在后续工作中继续被复用。

</SiteSection>

<SiteSection title="关键看点" description="Files 页最适合用来说明 AgentSmith 如何把文件从附件区升级成项目资产层。">

文件库、目录浏览和详情面板共同构成了一张很稳定的工作台。成员既可以手动上传资料，也可以让任务执行生成的产物自然回流到这里；而本地挂载又把 Web 端和本地工作目录接成了一条连续链路。这样的设计使 Files 不再只是“存文件的地方”，而是整个项目工作流的资产中枢。

</SiteSection>

<SiteSection title="关联模块" description="Files 与几乎所有使用面和运行链路都有直接关系。">

Files 会为 Chat 和 Notebook 提供上下文输入，也会承接智能体任务生成的 artifacts 与长期资料。再加上文件库创建、挂载说明和删除限制等页面，一整条资产管理链路就被完整串了起来。这也是 Files 在整个产品里存在感很强的原因，它几乎参与了所有重要工作流。

</SiteSection>
