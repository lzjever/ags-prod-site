<SiteSection title="界面概览" description="文件库页面用于查看项目文件、目录结构和挂载信息。">

Files 页面用于集中管理项目中的资料和产物。用户可以在这里查看文件库、浏览目录结构、检查文件详情，并在同一位置完成上传、挂载和引用。无论是原始资料、执行输入，还是任务生成的结果，都可以在这里被持续整理和保存。

当项目需要反复使用同一批资料，或需要把任务结果保留下来供后续工作继续使用时，这一页会变得尤为重要。

</SiteSection>

<SiteSection title="主要价值" description="文件库不仅保存资料，也承接任务产物和后续复用。">

资料管理和结果沉淀会在这里发生在同一套项目结构里。成员既可以手动上传文件，也可以让任务执行的产物自然回流到这里；同时，本地挂载功能又让浏览器中的文件库能够与本地工作目录协同使用。这样一来，文件不只是被保存下来，而是能够持续参与后续工作。

</SiteSection>

<SiteSection title="相关功能" description="文件库与对话、任务执行和本地挂载能力保持联动。">

Files 会为 Chat 和 Notebook 提供上下文输入，也会承接任务执行过程中生成的 artifacts。文件库创建、挂载信息和删除限制等相关页面，共同构成了一套完整的项目资产管理流程。

</SiteSection>
