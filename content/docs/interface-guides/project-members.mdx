<SiteSection title="界面概览" description="成员治理页面用于查看成员状态、加入申请和访问范围。">

成员治理页用于查看成员状态、处理加入申请、调整分组，并核对每位成员当前的有效访问范围。对于需要多人协作的项目，这一页能够帮助管理员持续掌握谁可以进入项目，以及进入后能够访问哪些能力。

</SiteSection>

<SiteSection title="主要价值" description="成员治理让项目的访问边界清晰地落到每一位成员身上。">

访问范围会在这里被落实到每一位成员身上。成员状态、加入申请和分组信息集中呈现后，项目权限会变得更容易理解，也更方便持续维护。

</SiteSection>

<SiteSection title="相关功能" description="成员治理与资源策略、审计记录及实际使用权限保持联动。">

成员分组会进一步影响资源策略，成员变更也会进入 Audit 留下记录，而最终生效的访问范围还会直接体现在 Chat、Notebook、Files 等页面的实际使用中。

</SiteSection>
