# 成员治理

<StatusBadge state="已实现" />

  preset="project-members-guide"

<SiteSection title="你会在这里做什么" description="这里用于管理成员、加入申请与访问状态。">

1. 搜索与查看成员当前状态。
2. 处理加入申请或调整分组。
3. 核对有效访问详情与权限来源。

</SiteSection>

<SiteSection title="关键看点" description="成员治理页最适合用来讲清楚项目级权限边界如何落到具体成员身上。">

1. 成员状态、加入申请和分组信息在同一界面里统一管理。
2. 权限并不是抽象概念，而是会直接影响模型、文件与任务的可访问范围。
3. 这让治理边界从配置层进入日常协作层。

</SiteSection>

<SiteSection title="关联模块" description="成员治理与资源策略、审计和实际使用行为天然相连。">

1. 成员分组会影响资源访问规则和有效权限。
2. 成员变更会进入 Audit，便于后续回看与排查。
3. 成员状态最终会影响 Chat、Notebook、Files 等模块的实际使用边界。

</SiteSection>
