<SiteSection title="这个页面的作用" description="成员治理页负责把项目访问边界落实到具体成员身上。">

项目一旦进入多人协作阶段，谁可以进入项目、进入后能使用哪些能力、成员变化是否已经生效，就都会成为持续需要管理的问题。成员治理页承担的，就是把这些边界清晰落实到每一位成员身上，并让管理员能够持续回看当前状态。

</SiteSection>

<SiteSection title="你通常会在这里做什么" description="查看成员状态、处理加入申请和确认访问范围，是这个页面最常见的动作。">

管理员通常会在这里查看项目当前有哪些成员、谁正在等待加入、各自属于什么分组，以及他们最终拥有怎样的有效访问范围。这一页的价值，不只是管理名单，而是把成员与权限边界持续对应起来，让后续使用结果更容易解释。

</SiteSection>

<SiteSection title="为什么它重要" description="成员边界不清晰，资源治理和使用解释都会变得困难。">

成员治理页会直接影响 Chat、Notebook、Files 和资源治理页面里的实际可用范围。成员变更也会进入审计记录，后续资源策略还可能围绕成员分组继续生效。也正因为如此，这一页并不是附属的人事页面，而是整个项目协作边界的基础。

</SiteSection>
