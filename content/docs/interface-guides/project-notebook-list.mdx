# Notebook 任务列表

<SiteSection title="界面概览" description="任务列表用于查看现有任务并继续创建新的任务。">

Notebook 列表页用于查看当前项目中的任务，并继续创建新的任务。列表会展示任务名称、绑定智能体和最近活动时间，帮助用户快速判断任务当前状态，并进入更具体的任务详情。

</SiteSection>

<SiteSection title="主要价值" description="任务列表让执行工作以持续对象的形式被管理和追踪。">

任务在这里以持续对象的形式被组织和追踪。用户可以围绕已有任务继续工作，而不必每次都从零开始重新组织一次执行过程。对于同时存在多个长期任务的项目来说，这一页尤其重要，因为它让成员能够快速判断哪些任务正在推进、哪些任务值得继续进入详情查看。

</SiteSection>

<SiteSection title="相关功能" description="任务列表与任务详情、执行能力和文件资产保持连续关系。">

从这里可以继续进入任务详情和创建任务页面，而任务本身又会与 Agents、Files 和 artifacts 保持连续关系，形成完整的执行链路。

</SiteSection>
