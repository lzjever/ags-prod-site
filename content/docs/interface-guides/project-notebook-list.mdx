<SiteSection title="你会在这里做什么" description="这里用于查看长期任务，并继续进入新的执行任务。">

Notebook 列表把复杂执行工作流先组织成稳定对象，再让用户进入更具体的任务详情。团队成员在这里看到的不是散落的执行记录，而是一组可以持续存在、持续更新、也方便继续协作的长期任务。这种组织方式，会让智能体工作流明显更像产品能力，而不是临时操作。

</SiteSection>

<SiteSection title="关键看点" description="这张页面最适合解释 Notebook 为什么不是一次性对话，而是长期任务系统。">

这张页面最有说服力的地方，在于任务被当作长期对象来管理。它们会携带智能体、最近活动、输入输出关系等关键信息，因此团队可以围绕这些对象持续推进工作，而不是每次都从零开始组织一次执行过程。

</SiteSection>

<SiteSection title="关联模块" description="Notebook 列表负责把长期任务组织起来，并把用户带向更具体的执行界面。">

从这里继续向下，用户会进入任务详情和创建任务动作，而整条链路又会和 Agents、Files、Artifacts 自然连在一起。这也是 Notebook 在白皮书叙事里很重要的一页，因为它很直观地展示了任务对象如何成为智能体工作的主载体。

</SiteSection>
