<SiteSection title="你会在这里做什么" description="这里用于查看长期任务，并继续进入新的执行任务。">

1. 展示长期任务集合。
2. 为任务详情页和创建任务动作导流。

</SiteSection>

<SiteSection title="关键看点" description="这张页面最适合解释 Notebook 为什么不是一次性对话，而是长期任务系统。">

1. 任务以列表形式存在，说明执行对象具备持续性和可回看性。
2. 任务与智能体、最近活动时间、输入输出关系都能在这里被管理。
3. 这让团队可以把复杂执行工作流组织成稳定对象，而不是临时操作。

</SiteSection>

<SiteSection title="关联模块" description="Notebook 列表负责把长期任务组织起来，并把用户带向更具体的执行界面。">

1. 向下连接任务详情页和创建任务动作。
2. 与 Agents、Files、Artifacts 形成连续执行链路。
3. 适合在白皮书中说明任务对象如何成为智能体工作的主载体。

</SiteSection>
