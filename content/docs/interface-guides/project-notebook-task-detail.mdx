<SiteSection title="你会在这里做什么" description="这里承接任务执行过程、运行状态与产物查看。">

这张页面把任务本身、执行过程、trace 和产物放到同一个视图里，用户不需要在终端、日志和临时目录之间来回切换，就能看到一次任务从开始到完成的完整过程。对实际使用者来说，这意味着复杂执行不再是“跑完就结束”的一次性动作，而是一个可以持续回看、继续协作的项目对象。

也正因为如此，Notebook 任务详情是理解 AgentSmith 差异化最直接的一张页面。它让智能体执行从个人工作流中的隐性过程，变成团队都能看懂、能复用、能追踪的正式能力。

</SiteSection>

<SiteSection title="关键看点" description="这张页面是整站里最适合体现 AgentSmith 差异化的一张执行界面。">

很多产品可以展示一次对话结果，但很少有产品能把任务状态、执行轨迹和输出产物组织成这样完整的业务视图。这里最有说服力的地方，不是界面里出现了多少信息，而是这些信息共同证明了一件事：智能体执行已经被纳入项目级工作流，而不是停留在个人工具链里。

如果要解释 AgentSmith 为什么不只是一个 Agent UI，这张页面通常会比抽象概念更有说服力，因为它直接展示了执行过程如何被组织、被沉淀、也被后续工作复用。

</SiteSection>

<SiteSection title="关联模块" description="任务详情页把输入、执行、产物与治理结果真正串联了起来。">

在这张页面背后，Files、Agents、Endpoints、Audit 和 Usage 并不是分散的概念。任务的输入可以来自项目文件、URL 或历史产物，执行依赖已接入的智能体和模型资源，产物又会继续沉淀回项目资产，并进入后续的观察与治理范围。也就是说，这不是一张孤立的任务页面，而是整条运行链路真正收口的地方。

</SiteSection>
