<SiteSection title="界面概览" description="任务详情页展示任务执行过程、运行状态和输出结果。">

任务详情页把任务内容、执行过程、trace 和产物放在同一个视图中。用户可以在一页内查看任务从开始到完成的全过程，而不需要在多个工具之间来回切换。对于需要持续跟踪执行状态和结果的场景，这种集中视图会明显提高可读性。

当一个任务需要被回看、继续推进，或交由其他成员接手时，这里也能提供足够完整的上下文。

</SiteSection>

<SiteSection title="主要价值" description="任务详情把执行过程、任务状态和输出结果集中到同一个视图中。">

这里不仅保留最终结果，也会完整保存状态变化、关键步骤和输出产物。团队可以据此判断任务进展，定位问题发生的位置，并继续复用已经生成的结果。

对于需要长期跟踪的任务来说，这种结构会比一次性对话更稳定，也更适合协作。

</SiteSection>

<SiteSection title="相关功能" description="任务详情与输入资料、执行能力和后续治理保持关联。">

任务的输入可以来自项目文件、URL 或历史产物，执行依赖项目中已接入的智能体和模型资源，输出结果又会继续沉淀到 Files 中。相关活动也会进入用量和审计记录，因此任务详情页天然连接着项目的执行、资产与管理链路。

</SiteSection>
