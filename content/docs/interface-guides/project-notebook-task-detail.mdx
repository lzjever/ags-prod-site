<SiteSection title="这个页面的作用" description="任务详情页承接一项复杂任务从开始到结束的完整记录。">

相比 Chat 更适合快速交互，Notebook 任务详情页更适合承接需要持续推进的复杂工作。一个任务在这里会同时保留输入背景、消息过程、trace、运行状态和产物文件，因此用户不需要在多个工具之间来回切换，就能看到任务是怎样一步步运行下去的。

</SiteSection>

<SiteSection title="你通常会在这里做什么" description="查看过程、判断进展、回看结果和继续推进，是这个页面最常见的动作。">

用户通常会在这里观察任务当前运行到了哪一步、有没有新的消息或 trace、是否已经生成结果文件，以及接下来是否需要继续补充输入或发起下一轮操作。对于需要长期跟踪的工作来说，这种集中视图会明显降低理解成本，因为任务的关键事实都留在同一个位置。

</SiteSection>

<SiteSection title="为什么这个页面对团队很重要" description="它把复杂执行从一次调用变成了一份可以回看和接手的任务记录。">

一项复杂任务真正有价值的，不只是最后跑出了什么结果，还包括它基于什么输入、执行过程中发生了什么、留下了哪些产物，以及后续成员能否继续接手。任务详情页把这些内容持续保留在一起，使复杂执行不再停留在瞬时结果里，而能真正进入团队协作和后续复用。

</SiteSection>
