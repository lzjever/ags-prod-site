<SiteSection title="界面概览" description="项目总览用于帮助用户确认当前项目状态并进入主要功能。">

进入项目后，用户首先需要确认当前项目的基本信息，并快速找到接下来要使用的主要功能。项目总览页承担的正是这个作用。它把项目名称、说明和关键功能集中展示，帮助成员在最短时间内进入当前工作上下文。

从这里继续进入 Chat、Notebook、Files 或管理页面时，整个操作路径会保持一致。无论是发起对话、查看任务，还是进入资源和成员管理，用户都不需要重新建立上下文。

</SiteSection>

<SiteSection title="主要价值" description="项目总览把日常使用和治理能力统一到同一个项目视图中。">

项目总览页的价值在于，它把一个项目需要的核心操作放在同一个视图中。普通成员可以从这里进入日常工作，管理员也可以从这里继续进入资源、成员和审计管理。对团队来说，这种统一视图能够减少切换成本，并让项目边界始终保持清晰。

项目在这里不是一个简单名称，而是组织任务、资料、模型资源和访问范围的基本单位。后续所有工作都会围绕这个单位展开。

</SiteSection>

<SiteSection title="相关功能" description="从项目总览可以继续进入项目中的主要工作模块。">

从项目总览出发，用户可以进入 Chat、Notebook、Files 和 Agents 等工作页面，也可以继续进入 Endpoints、资源策略、用量与审计等管理页面。它既是进入日常工作的起点，也是理解整个项目结构的总览页。

</SiteSection>
