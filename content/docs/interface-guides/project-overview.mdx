<SiteSection title="你会在这里做什么" description="这是进入项目后的第一屏，用于确认上下文并继续展开工作。">

1. 帮助用户确认当前项目上下文。
2. 作为进入 Chat、Notebook、Files 和治理能力的统一起点。
3. 避免 Workspace Home 和 Project Overview 演化为伪大盘。

</SiteSection>

<SiteSection title="关键看点" description="这张页面最适合用来解释项目为什么是 AgentSmith 的核心容器。">

1. Use 面与 Govern 面在同一项目语境中展开，而不是分裂成两套入口。
2. 用户能快速确认当前项目的角色、范围和可继续操作的功能。
3. 这里决定了后续聊天、任务、文件和治理动作都围绕同一上下文发生。

</SiteSection>

<SiteSection title="关联模块" description="项目总览不是终点，而是整个产品主线的分发中心。">

1. 向下连接 Chat、Notebook、Files 与 Agents 的使用面。
2. 同时连接 Endpoints、Policy、Usage、Audit 等治理能力。
3. 是解释 `Project` 在整体架构中地位时最重要的落点界面。

</SiteSection>
