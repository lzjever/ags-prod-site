<SiteSection title="这个页面的作用" description="项目总览是进入项目后的第一层工作台。">

进入一个项目后，成员首先需要确认自己当前所在的项目、理解这个项目承接什么工作，并快速找到后续最常用的入口。项目总览页承担的正是这个作用。它把项目基础信息与主要功能入口放在同一视图中，让用户在最短时间内进入正确上下文，而不必先在多个侧栏和页面之间来回摸索。

</SiteSection>

<SiteSection title="你通常会在这里做什么" description="这个页面更像一张项目地图，而不是具体操作页。">

用户通常会在这里确认项目名称、了解项目说明、判断接下来应该进入对话、任务、文件还是资源管理。对于普通成员来说，这里是开始日常工作的起点；对于管理员来说，这里也是继续进入成员、资源、策略和审计页面的入口。因此，这一页并不负责完成某一项具体任务，而是负责告诉你“这个项目现在能做什么，下一步该去哪里”。

</SiteSection>

<SiteSection title="为什么它重要" description="项目总览决定了团队能否把整个系统看成同一个项目，而不是很多零散后台。">

如果项目中的聊天、任务、文件、智能体和管理入口彼此割裂，成员很容易把产品理解成很多独立后台。项目总览页的价值，就在于它把这些入口重新放回同一个项目上下文中，让团队更容易理解后续所有动作都会围绕同一个 Project 继续发生。

</SiteSection>
