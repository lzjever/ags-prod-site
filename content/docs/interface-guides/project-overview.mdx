<SiteSection title="你会在这里做什么" description="这是进入项目后的第一屏，用于确认上下文并继续展开工作。">

进入项目后，用户首先需要确认自己身处哪个业务上下文、接下来该往哪里继续，以及这个项目当前承载了哪些核心能力。项目总览承担的正是这个角色。它不是一个为了“放很多卡片”而存在的大盘，而是帮助团队快速进入工作状态的正式起点。

从这里继续进入 Chat、Notebook、Files 或治理页面，整个体验会显得非常连贯，因为这些能力并不是后来拼接在一起的，而是天然围绕同一个项目边界展开。

</SiteSection>

<SiteSection title="关键看点" description="这张页面最适合用来解释项目为什么是 AgentSmith 的核心容器。">

这张页面最重要的价值，在于把“使用”与“治理”放回同一个项目语境里。对团队成员来说，它说明所有后续动作都发生在同一业务边界内；对管理者来说，它也明确了后续的资源控制、成员协作和审计回看，都不会脱离这个上下文单独存在。

换句话说，项目不是一个简单的目录层级，而是 AgentSmith 组织 AI 工作、权限边界和执行记录的核心容器。

</SiteSection>

<SiteSection title="关联模块" description="项目总览不是终点，而是整个产品主线的分发中心。">

从项目总览出发，用户很自然地会走向两条主线：一条是 Chat、Notebook、Files、Agents 这样的日常工作入口，另一条是 Endpoints、Policy、Usage、Audit 这样的治理与管理入口。也正因为它同时连接这两条主线，这张页面成为解释 AgentSmith 整体结构时最有说服力的落点之一。

</SiteSection>
