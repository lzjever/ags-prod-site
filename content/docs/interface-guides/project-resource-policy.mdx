<SiteSection title="你会在这里做什么" description="这里用于设置访问规则、限流和费用约束。">

1. 调整访问模式与限额。
2. 查看 subject allow-list 与 explainability。
3. 与 Usage / Audit 形成治理闭环。

</SiteSection>

<SiteSection title="关键看点" description="资源策略页最适合说明治理不是抽象概念，而是会直接影响资源使用方式。">

1. 访问模式、限额和主体 allow-list 都在这里被明确配置。
2. 用户不仅能看到规则，还能理解为什么某条规则会生效。
3. 这让治理结果在日常使用中具备可解释性，而不是黑箱限制。

</SiteSection>

<SiteSection title="关联模块" description="策略配置最终会在模型使用、用量观察和审计回看中持续生效。">

1. 作用对象来自 Endpoint。
2. 结果会在 Usage 中被感知，在 Audit 中被回看。
3. 对 Chat、Notebook 与 Agents 的资源使用方式都有直接影响。

</SiteSection>
