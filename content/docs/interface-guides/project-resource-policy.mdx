# 资源策略

<SiteSection title="你会在这里做什么" description="这里用于设置访问规则、限流和费用约束。">

资源策略页决定了一件很现实的事情：谁可以使用哪些资源、以什么方式使用、在什么限制内使用。无论是访问模式、限额，还是面向具体主体的允许范围，这些规则都会直接影响团队的日常体验，因此它不是一个只有管理员才会在意的抽象页面，而是企业 AI 可控落地的关键一环。

更重要的是，这里的规则不是写完就结束，它们会继续体现在后续的用量观察、异常排查和审计回看之中。

</SiteSection>

<SiteSection title="关键看点" description="资源策略页最适合说明治理不是抽象概念，而是会直接影响资源使用方式。">

这张页面真正有说服力的地方，在于规则不仅被展示出来，还具备解释性。用户不仅知道“不能这样用”或“只能到这里为止”，也更容易理解为什么规则会这样生效。对于企业场景来说，这种可解释性要比单纯的限制更重要，因为它决定了治理是否会被团队接受并长期执行。

</SiteSection>

<SiteSection title="关联模块" description="策略配置最终会在模型使用、用量观察和审计回看中持续生效。">

资源策略围绕 Endpoint 生效，它带来的影响会在 Usage 中被感知，在 Audit 中被回看，也会直接改变 Chat、Notebook 和 Agents 的实际使用方式。也正因为如此，策略不是离业务很远的一层，而是整套产品结构中最贴近“可控使用”这个目标的一页。

</SiteSection>
