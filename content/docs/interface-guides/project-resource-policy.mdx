<SiteSection title="界面概览" description="资源策略页面用于设置访问规则、限额和费用约束。">

资源策略页用于设置访问规则、限额和费用约束。管理员可以在这里定义谁可以使用哪些资源、在什么限制内使用，以及不同主体对应的允许范围。

这些规则会直接影响项目中的日常使用体验，并持续体现在后续的用量观察和审计记录中。

</SiteSection>

<SiteSection title="主要价值" description="资源策略让项目中的资源使用规则清晰、可解释并可执行。">

这里会把资源使用规则明确展示出来，并让后续限制、限流和费用控制有统一依据。对于需要稳定管理模型资源的团队来说，这一点尤其重要。

</SiteSection>

<SiteSection title="相关功能" description="资源策略与模型使用、用量观察和审计回看保持连续关系。">

资源策略围绕 Endpoint 生效，其影响会在 Usage 中被观察，在 Audit 中被回看，也会直接改变 Chat、Notebook 和 Agents 的实际使用范围。

</SiteSection>
