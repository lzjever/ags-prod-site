# 项目设置

<StatusBadge state="已实现" />

  preset="project-settings-guide"

<SiteSection title="你会在这里做什么" description="这里用于维护项目基础信息、可见性与加入策略。">

项目设置页负责维护一个项目最基本、也最长期有效的边界条件。名称、描述、可见性和加入策略都会在这里被统一维护，这些信息看似基础，却会持续影响成员协作、访问范围和整个项目生命周期。

</SiteSection>

<SiteSection title="关键看点" description="项目设置页最适合说明项目并不是临时工作空间，而是有明确生命周期和治理边界的正式对象。">

这张页面最适合用来说明，项目在 AgentSmith 里并不是一次性工作空间，而是长期存在的正式容器。它承载的不只是当前任务，还承载了成员协作、资源治理和后续回看需要依赖的边界信息。

</SiteSection>

<SiteSection title="关联模块" description="项目设置与成员治理、资源治理和审计回看共同构成项目管理层。">

项目设置会直接影响成员访问范围，变更又会进入 Audit 留下依据，因此它始终和成员治理、资源治理以及审计回看连在一起。对解释项目生命周期管理来说，这是一张很重要的支点页面。

</SiteSection>
