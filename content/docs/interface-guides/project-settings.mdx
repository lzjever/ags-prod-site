<SiteSection title="你会在这里做什么" description="这里用于维护项目基础信息、可见性与加入策略。">

1. 查看项目基础配置。
2. 修改说明、可见性或相关设置。

</SiteSection>

<SiteSection title="关键看点" description="项目设置页最适合说明项目并不是临时工作空间，而是有明确生命周期和治理边界的正式对象。">

1. 项目名称、描述、可见性和加入策略都在这里集中维护。
2. 这些基础配置会直接影响成员协作和治理边界。
3. 页面体现了项目作为长期容器的正式性，而不是一次性会话空间。

</SiteSection>

<SiteSection title="关联模块" description="项目设置与成员治理、资源治理和审计回看共同构成项目管理层。">

1. 可见性和加入策略会影响成员访问范围。
2. 设置变更会进入 Audit，方便后续回看。
3. 是解释项目生命周期管理时的重要界面支点。

</SiteSection>
