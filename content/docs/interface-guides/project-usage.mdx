# 用量与限额

<ScreenshotFrame

<SiteSection title="你会在这里做什么" description="这里用于查看剩余额度、支出与使用趋势。">

Usage 页面把“资源到底用了多少、还剩多少、变化趋势如何”这些问题变得一目了然。对一线成员来说，它能帮助他们理解当前的资源状态；对管理者来说，它提供了是否需要调整策略、限制或预算的第一手依据。相比只在后台被动设限，这种可见的用量反馈会让治理更容易被团队理解和接受。

</SiteSection>

<SiteSection title="关键看点" description="用量页最适合说明治理不仅是限制，还包括对资源消耗的持续理解。">

这张页面最好的地方，是把治理从“只有出问题时才会想到的后台动作”变成一种日常感知。用户可以按 Endpoint 理解不同模型资源的消耗差异，也能更直观地看到使用趋势和剩余额度。这样一来，资源管理就不再只是少数管理员的职责，而会自然进入团队的日常判断。

</SiteSection>

<SiteSection title="关联模块" description="Usage 是资源治理链路中最贴近日常使用体验的一层。">

Usage 观察的是 Endpoint，限制条件来自 Resource Policy，而异常波动和关键变化又能继续在 Audit 中被追溯。它正好处在治理链路中最贴近日常使用体验的一层，把资源管理从规则层真正拉到了业务现场。

</SiteSection>
