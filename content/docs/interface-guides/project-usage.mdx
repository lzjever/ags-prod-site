# 用量与限额

<SiteSection title="这个页面的作用" description="Usage 页面负责让资源消耗变得持续可见。">

当项目开始稳定使用模型资源后，团队就需要知道当前消耗到了什么程度、剩余额度还有多少、最近的变化是否正常。Usage 页面承担的，就是把这些信息直接展示出来，让资源状态不再隐藏在后台记录中。它不是简单的数字堆叠，而是围绕 Endpoint 汇总出来的日常观察面。

</SiteSection>

<SiteSection title="你通常会在这里看什么" description="按 Endpoint 查看消耗、额度和趋势，是这个页面最常见的使用方式。">

管理员或相关成员通常会在这里按不同 Endpoint 查看当前消耗情况、剩余额度和趋势变化，并据此判断是否需要调整资源策略、预算安排或后续调用方式。比如某个模型最近消耗是否突然升高，某项额度是不是快用完了，最近 30 天趋势是不是开始偏离常态，是否需要提前收紧限制或切换资源。

</SiteSection>

<SiteSection title="为什么它重要" description="如果看不到真实消耗，很多管理动作就只能靠猜。">

Usage 页面的价值，在于它让资源状态直接可见。资源策略决定了限制边界，Audit 解释了关键变化，而 Usage 则告诉团队这些资源在真实使用中到底发生了什么。只有把这三层接起来，团队才不会只知道“好像变多了”或“似乎被限制了”，而是能看清到底是哪一个 Endpoint 在消耗、变化从什么时候开始、后面应该去哪个页面继续查。

</SiteSection>
