<SiteSection title="这个页面的作用" description="Usage 页面负责让资源消耗变得持续可见。">

当项目开始稳定使用模型资源后，团队就需要知道当前消耗到了什么程度、剩余额度还有多少、最近的变化是否正常。Usage 页面承担的，就是把这些信息直接展示出来，让资源状态不再隐藏在后台记录中。

</SiteSection>

<SiteSection title="你通常会在这里看什么" description="按 Endpoint 查看消耗、额度和趋势，是这个页面最常见的使用方式。">

管理员或相关成员通常会在这里按不同 Endpoint 查看当前消耗情况、剩余额度和趋势变化，并据此判断是否需要调整资源策略、预算安排或后续调用方式。对于持续使用 AI 的团队来说，这一页相当于资源健康状况的日常观察窗口。

</SiteSection>

<SiteSection title="为什么它重要" description="如果看不到真实消耗，治理就很难进入长期运行。">

Usage 页面的价值，在于它让资源状态直接可见。资源策略决定了限制边界，Audit 解释了关键变化，而 Usage 则告诉团队这些资源在真实使用中到底发生了什么。只有把这三层接起来，团队才既能限制资源，也能理解资源。

</SiteSection>
