# 工作区登录

<SiteSection title="这个页面的作用" description="工作区登录页负责在进入业务空间前完成身份确认。">

当用户已经选择了目标工作区后，系统还需要确认当前身份是否可以进入这个业务空间，并以一致方式建立后续会话。工作区登录页承担的，就是把这一步稳定地放在进入项目之前完成。

</SiteSection>

<SiteSection title="你通常会在这里做什么" description="确认身份并进入目标工作区，是这个页面最核心的动作。">

用户会在这里完成进入工作区所需的登录动作，随后再进入项目列表和具体页面。对多工作区场景来说，这一页的意义不在于“再登录一次”，而在于确保接下来的所有项目操作都确实发生在当前业务空间内。

</SiteSection>

<SiteSection title="为什么它重要" description="工作区身份确认越清晰，后续协作和权限解释就越稳定。">

工作区登录页会让身份确认发生在一个明确的业务空间里，而不是在进入项目之后再让用户猜测自己到底属于哪一个范围。这样一来，后续看到的项目、成员关系和资源边界也会更容易保持一致。

</SiteSection>
