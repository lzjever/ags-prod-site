# 工作区项目列表

<SiteSection title="你会在这里做什么" description="这里用于浏览项目、搜索项目并创建新项目。">

工作区项目列表承接的是团队最日常的一类动作: 找到项目、进入项目，或者创建新的项目主线。它让成员在同一工作区上下文中完成这些操作，而不是在多个入口间来回切换，因此整个层级关系会显得非常自然。

</SiteSection>

<SiteSection title="关键看点" description="工作区项目列表最适合说明团队如何在同一工作区下组织多个项目。">

这张页面很适合用来解释 Workspace 与 Project 的关系。用户会很直观地看到，同一个业务空间下可以组织多个项目，而每个项目又都会承接一条独立但结构一致的 AI 工作主线。

</SiteSection>
