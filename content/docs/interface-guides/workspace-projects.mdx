# 工作区项目列表

<StatusBadge state="已实现" />

<ScreenshotFrame
  preset="workspace-projects-guide"
  caption="工作区项目列表让成员在同一上下文中查看项目、搜索项目并创建新项目。"
/>

<SiteSection title="这个页面的作用" description="工作区项目列表页负责承接同一业务空间中的多个项目。">

进入工作区之后，用户需要先看到这个业务空间里有哪些项目、自己可以进入哪些项目，以及是否需要新建项目来承接新的工作。工作区项目列表页承担的，就是把这些项目统一呈现在同一个上下文中。

</SiteSection>

<SiteSection title="你通常会在这里做什么" description="浏览、搜索和创建项目，是这个页面最常见的动作。">

用户会在这里浏览已有项目、搜索需要进入的项目，并在需要时发起创建新项目。对同时维护多个项目的团队来说，这一页不仅仅是列表视图，更是统一管理项目入口的位置。

</SiteSection>

<SiteSection title="为什么它重要" description="没有这一层统一入口，多项目协作会很快变得零散。">

工作区项目列表页让同一业务空间中的多个项目保持一致的组织方式。成员因此更容易知道自己当前处在哪个范围内、有哪些项目正在进行，以及新的工作应当落到哪个项目中继续推进。

</SiteSection>
