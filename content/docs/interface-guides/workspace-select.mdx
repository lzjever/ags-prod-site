# 工作区选择

<SiteSection title="这个页面的作用" description="工作区选择页负责先确定用户接下来要进入哪一个业务空间。">

当用户拥有多个可访问工作区时，系统需要先确认当前要进入哪一个业务空间。工作区选择页承担的，就是在所有项目和具体功能出现之前，先把这层边界确定下来。这样后续看到的项目列表、成员关系和资源范围，都会与这一次选择保持一致。

</SiteSection>

<SiteSection title="你通常会在这里做什么" description="选择目标工作区，是这个页面唯一但非常关键的动作。">

用户会在这里查看自己当前可以访问哪些工作区，并进入其中一个继续后续工作。虽然界面动作很简单，但这一步的意义很大，因为它决定了接下来所有项目、资源和协作关系都属于哪一个业务空间。

</SiteSection>

<SiteSection title="为什么它重要" description="如果没有先确认工作区边界，后续项目上下文就会变得模糊。">

工作区不是一个普通分组，而是业务空间的上层边界。它帮助系统先区分不同组织、不同团队或不同业务域，再让项目在这个边界内继续组织。因此，工作区选择页虽然是入口页，却直接决定了后续上下文是否清晰。

</SiteSection>
