<SiteSection title="你会在这里做什么" description="这里用于选择工作区，并进入对应的业务上下文。">

用户进入系统后的第一件事，不是直接去操作项目，而是先确认自己要进入哪个工作区。这一步让业务空间的边界先被确定下来，也让后续看到的项目、权限和协作关系都具备清晰前提。

</SiteSection>

<SiteSection title="关键看点" description="工作区选择页最适合说明系统级入口与业务级入口之间的层级关系。">

这张页面的意义并不只是“选一个列表项”，而是在产品结构上明确: Workspace 是真实租户边界，而不是一个装饰性的分组概念。正因为有这层上位上下文，项目级使用和治理才能显得自然且稳定。

</SiteSection>
