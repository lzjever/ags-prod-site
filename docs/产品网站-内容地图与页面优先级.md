# AgentSmith 产品网站内容地图与页面优先级

更新日期：2026-03-17  
状态：`draft-v0.1`

## 1. 文档目的

本文档用于确定 `ags-prod-site` 的第一版站点骨架，包括：

1. 一级导航结构
2. 各栏目职责
3. 页面清单
4. 页面优先级
5. 每类页面的主视觉形式

它是后续首页设计、信息架构实现、文档导入和视觉组件设计的直接依据。

## 2. 站点总体判断

AgentSmith 产品网站不是单一类型站点，而是三种站点能力的组合：

1. `产品白皮书站`
2. `功能说明站`
3. `市场推广与演示站`

因此站点结构不能只按 PRD 文件树组织，也不能只按截图目录组织。

更合理的做法是：

1. 用首页和专题页承接产品叙事与推广表达。
2. 用文档型栏目承接白皮书、架构和模块说明。
3. 用证据型栏目承接截图、页面说明和功能索引。

## 3. 一级导航建议

第一版一级导航建议固定为以下 7 个栏目：

1. `首页`
2. `产品白皮书`
3. `产品架构`
4. `核心能力`
5. `智能体运行专题`
6. `界面与功能说明`
7. `版本与资料`

这是当前推荐的“对外可讲、对内可维护”的最小完整结构。

## 4. 一级栏目定义

### 4.1 首页

首页负责：

1. 一句话定义 AgentSmith
2. 强化核心价值和差异化
3. 用高质量视觉快速建立产品气质
4. 为后续阅读提供清晰入口

首页不是目录页，也不是白皮书摘要页，而是站点的主叙事入口。

### 4.2 产品白皮书

该栏目负责：

1. 产品摘要
2. 产品定位
3. 目标与范围边界
4. 核心价值
5. 成熟度判断
6. 路线方向

该栏目偏高层叙事，适合售前、产品和管理层快速理解。

### 4.3 产品架构

该栏目负责解释：

1. `System / Workspace / Project / Use / Govern` 结构
2. 核心对象与对象关系
3. 关键任务流
4. 技术架构高层图
5. Use 面与 Govern 面如何联动

该栏目偏“讲清楚系统怎么组织”。

### 4.4 核心能力

该栏目负责按模块解释产品能力，包括：

1. Chat
2. Notebook
3. Files
4. Agents
5. Endpoints
6. Resource Policy
7. Credentials
8. Members
9. Usage
10. Audit
11. Alerts / Settings（视内容密度决定是否合并）

该栏目偏“模块理解与价值拆解”。

### 4.5 智能体运行专题

这是站点的差异化重点栏目，单独承接：

1. 通用智能体为何难以落地
2. AgentSmith 如何把通用智能体升级为团队可用能力
3. Notebook、Files、Agents、Skills、Credential Files、Task Workspace 如何连成一条链
4. 托管执行、受控运行、产物沉淀、文件持久化的产品意义

这个栏目不能被分散到各个模块页里稀释掉。

### 4.6 界面与功能说明

该栏目负责：

1. 基于最新截图目录展示真实页面
2. 对每个关键页面给出功能说明与用户操作说明
3. 形成外部演示和内部对齐时可直接引用的“页面证据库”

这个栏目既是功能索引，也是视觉资产承载区。

### 4.7 版本与资料

该栏目负责：

1. 版本基线
2. 已实现 / 部分实现 / 规划中状态说明
3. 参考资料说明
4. 文档来源与更新机制
5. 可选的路线图与演进记录

该栏目偏“可信度和维护性”。

## 6. 页面类型与主视觉形式

为了保证全站有统一语言但不单调，页面视觉形式建议按类型区分。

### 6.1 首页与专题页

适合使用：

1. 大尺寸 hero
2. 截图拼贴与分层动画
3. 滚动联动流程图
4. 模块亮点卡片
5. 大段叙事与可视化穿插

目标是建立品牌感、可信感和高级感。

### 6.2 白皮书与架构页

适合使用：

1. 结构化大标题
2. 关键观点卡片
3. 架构图和流程图
4. 截图局部作为证据插图
5. 状态标记和结论型摘要

目标是读起来清楚、严谨、可引用。

### 6.3 模块能力页

适合使用：

1. 模块头图
2. 功能价值摘要
3. 与其他模块的关系图
4. 关键截图高亮
5. 用户故事和任务流组合展示

目标是让用户快速理解“这个模块为什么重要”。

### 6.4 页面证据页

适合使用：

1. 大截图
2. 局部放大与标注
3. 路由、角色、操作步骤信息条
4. 对话框与主页面成组展示

目标是形成“可演示、可说明、可比对”的证据库。

## 7. 首页信息架构建议

首页建议采用以下顺序：

1. `Hero`
   直接给出一句话定义和高冲击截图组合。
2. `为什么需要 AgentSmith`
   解释企业 AI 使用与通用智能体落地的共同难题。
3. `双主线定位`
   企业级 AI 控制平面 + 通用智能体运行环境。
4. `核心结构`
   System / Workspace / Project / Use / Govern。
5. `四大亮点`
   Chat、Notebook、Files、Governance。
6. `智能体运行链路`
   用视觉化流程讲 Notebook + Files + Agents。
7. `真实界面证据`
   精选 4-6 张关键页面截图。
8. `继续阅读入口`
   白皮书、架构、能力、截图说明。

## 9. 页面分组与实现顺序建议

从开发效率和站点完整性出发，建议按以下顺序推进实现：

1. `组 A：站点骨架`
   首页、文档布局、导航、主题变量、基础组件。
2. `组 B：主叙事页面`
   白皮书、架构总览、智能体运行专题。
3. `组 C：核心能力页面`
   Chat、Notebook、Files、Agents、Governance 总览。
4. `组 D：截图证据页面`
   页面索引和首批关键页面说明。
5. `组 E：补全页面`
   其余治理页、对话框页、版本资料页。

## 10. 站点导航建议

为了平衡视觉站点与文档站体验，建议采用：

1. 顶部主导航承接 7 个一级栏目。
2. 文档区侧边栏按栏目展开二级页面。
3. 首页和专题页保留更自由的视觉布局，不强行套成文档模板。
4. 页面证据区允许提供截图索引和按模块筛选能力。

## 11. 当前结论

当前第一版内容地图已经足够支撑下一阶段实施。

最重要的结论有三条：

1. 首页、白皮书、架构、能力、专题、证据、版本资料这 7 个一级栏目是当前最优骨架。
2. `智能体运行专题` 必须独立成栏，不能被拆散到普通模块页里。
3. 页面实现必须分优先级推进，先确保主叙事成立，再补足全量证据和细节页面。
