# AgentSmith 产品网站内容状态治理与更新规则

更新日期：2026-03-17  
状态：`draft-v0.1`

## 1. 文档目的

本文档用于规范产品网站中能力状态、事实来源和内容更新方式，避免站点内容逐步偏离 `agentsmith` 的真实产品状态。

## 2. 状态定义

产品网站统一使用以下三种状态：

1. `已实现`
2. `部分实现`
3. `规划中`

这三种状态必须与参考 PRD 语义一致。

## 3. 状态判断依据

判断能力状态时，优先依据以下来源：

1. `references/prd-v1/`
2. `references/product-docs/latest/`
3. `../agentsmith` 中可验证的页面、代码、测试、合同与开发文档

如果多个来源发生冲突，默认处理顺序为：

1. 事实实现与可验证页面
2. 当前 PRD 明确状态
3. 站点已有表述

## 4. 状态使用规则

### 4.1 已实现

仅在以下情况可使用：

1. 已有真实页面或稳定功能链路
2. 可通过截图、代码或测试佐证
3. 不是只有规划说明而无实现证据

### 4.2 部分实现

适用于：

1. 主链路存在，但闭环不完整
2. 前台可见，但后端基础设施未完全收口
3. 有明确产品方向，但当前能力仍有限制

### 4.3 规划中

适用于：

1. 已有明确方向和文档表达
2. 当前尚无完整实现证据
3. 不应被首页主卖点当作已落地能力宣传

## 5. 首页卖点约束

首页和主专题页可以介绍未来方向，但必须遵守：

1. 首页主卖点优先使用 `已实现` 或有强证据支撑的 `部分实现` 能力。
2. `规划中` 能力不得作为当前核心承诺来宣传。
3. 如果必须提到未来方向，应明确标识其状态。

## 6. 更新触发条件

以下情况发生时，应检查站点内容是否需要更新：

1. PRD 发生版本更新
2. `product-docs` 最新截图发生明显变化
3. 主产品页面结构或功能发生明显变化
4. 关键专题的状态从 `部分实现` 变为 `已实现`
5. 路线图表达发生变化

## 7. 更新流程建议

每次更新建议按以下顺序进行：

1. 执行 `./scripts/sync-reference-sources.sh`
2. 核对最新截图和说明
3. 核对 PRD 与代码事实
4. 更新站点页面文案与状态标记
5. 在 `版本与资料` 栏目中同步更新相关说明

## 8. 漂移风险提示

以下是最常见的内容漂移风险：

1. 把未来路线图写成当前能力
2. 用旧截图解释新结构
3. 页面功能已经变化，但站点说明未更新
4. 为了营销效果弱化能力边界

## 9. 当前结论

产品网站必须以“可信优先”而不是“夸张优先”为原则。

站点如果失去状态治理，就会很快变成一套漂亮但不可靠的介绍页，因此本规则必须作为长期维护基线。
