# AgentSmith 产品网站受众分层与阅读路径

更新日期：2026-03-17  
状态：`draft-v0.1`

## 1. 文档目的

本文档用于明确产品网站主要面向哪些读者，以及不同读者进入站点后应如何被引导阅读。

目标不是把所有人都放进同一条路径，而是让不同角色都能快速进入最适合自己的内容链路。

## 2. 总体原则

AgentSmith 产品网站必须同时服务多类读者，但不应试图让所有内容在同一页面里同时满足所有人。

正确做法是：

1. 首页统一回答“这是什么、为什么重要、凭什么可信”。
2. 后续阅读路径按角色分流。
3. 每类读者都能在 2 到 3 次点击内进入自己最关心的内容。

## 3. 核心读者分层

### 3.1 管理者 / 决策者

关注点：

1. 这是什么产品
2. 它解决什么组织级问题
3. 为什么它不是普通 AI 工具
4. 当前成熟度到什么阶段

推荐阅读路径：

1. `首页`
2. `产品白皮书 / 产品摘要`
3. `产品白皮书 / 定位与范围边界`
4. `产品白皮书 / 核心价值`
5. `产品白皮书 / 成熟度与路线方向`

他们不需要先看大量页面细节。

### 3.2 售前 / 市场 / 交付

关注点：

1. 如何用清晰叙事介绍产品
2. 有哪些可直接拿来演示的亮点
3. 哪些界面最能证明能力真实存在
4. 哪些专题最能体现差异化

推荐阅读路径：

1. `首页`
2. `智能体运行专题`
3. `核心能力 / Governance 总览`
4. `界面与功能说明 / 页面索引总览`
5. 关键页面证据页

### 3.3 产品 / 设计

关注点：

1. 产品定位和范围边界
2. 信息架构和模块关系
3. 用户故事和任务流
4. 页面与叙事如何统一

推荐阅读路径：

1. `首页`
2. `产品白皮书`
3. `产品架构`
4. `核心能力`
5. `界面与功能说明`

### 3.4 研发 / 架构 / 测试

关注点：

1. 站点表达是否与真实实现一致
2. 技术架构高层说明
3. 页面证据和功能状态是否可信
4. 哪些能力已实现、部分实现、规划中

推荐阅读路径：

1. `产品架构`
2. `核心能力`
3. `智能体运行专题`
4. `界面与功能说明`
5. `版本与资料`

### 3.5 潜在客户 / 外部合作方

关注点：

1. 这个产品到底能做什么
2. 为什么它比常见 AI 平台更适合真实团队使用
3. 是否有真实界面和任务场景支撑

推荐阅读路径：

1. `首页`
2. `产品白皮书 / 产品摘要`
3. `核心能力 / Notebook`
4. `智能体运行专题 / Notebook + Files + Agents 运行链路`
5. `界面与功能说明 / 项目总览`

## 4. 首页分流要求

首页应至少为以下三类读者提供明显入口：

1. `先读产品定位`
2. `先看核心能力`
3. `先看真实界面`

如果后续首页支持角色分流组件，建议使用：

1. `给管理者`
2. `给售前与交付`
3. `给设计与研发`

## 5. 内容编排原则

不同读者层级对页面深度的要求不同，因此：

1. 首页和白皮书页优先说结论和价值。
2. 架构页优先说关系和边界。
3. 模块页优先说问题、价值、使用方式。
4. 证据页优先说真实页面、功能说明和操作步骤。

## 6. 当前结论

本网站不是“一个页面讲给所有人听”，而是“一个统一入口，随后按读者角色分流”。

后续所有首页设计、导航设计和 CTA 设计都应以此为准。
