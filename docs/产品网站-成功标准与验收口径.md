# AgentSmith 产品网站成功标准与验收口径

更新日期：2026-03-17  
状态：`draft-v0.1`

## 1. 文档目的

本文档用于定义产品网站第一阶段和后续阶段的成功标准，避免站点建设只追求“页面做完了”或“视觉很炫”，却无法判断是否真正达成目标。

## 2. 成功定义

AgentSmith 产品网站的成功，不以页面数量衡量，而以以下结果衡量：

1. 读者能快速理解产品定位。
2. 读者能清楚区分 AgentSmith 的双主线。
3. 关键能力能通过真实界面和任务流被可信呈现。
4. 售前、交付、产品、研发都能直接使用站点内容。
5. 站点结构可持续维护，不因后续迭代而失控。

## 3. 第一阶段成功标准

### 3.1 认知标准

第一阶段至少应做到：

1. 读者在 10 秒内能理解 AgentSmith 是什么类型的产品。
2. 读者在 30 秒内能理解它不是普通模型接入面板。
3. 读者在 3 分钟内能理解“企业级 AI 控制平面 + 通用智能体运行环境”双主线。

### 3.2 内容标准

1. 站点必须覆盖产品白皮书、架构、核心能力、专题、界面证据和版本资料六类核心内容。
2. P0 页面必须能独立讲通主叙事。
3. 所有首页主卖点都必须能在后续页面中找到对应证据。

### 3.3 可信度标准

1. 关键能力说明必须基于 PRD、截图和代码事实。
2. 页面说明必须与最新截图一致。
3. 已实现、部分实现、规划中的状态表达必须一致。

### 3.4 视觉标准

1. 站点视觉必须与 `agentsmith` 主产品气质协调。
2. 首页必须具备显著的产品站品质，而不是默认文档站外观。
3. 动效和视觉增强必须服务理解，而不是制造噪音。

## 4. 站点验收口径

### 4.1 首页验收

首页通过验收，至少应满足：

1. 有稳定的一句话定义。
2. 有清晰的双主线表达。
3. 有清晰的下一步阅读入口。
4. 有真实截图支撑。
5. 整体视觉明显高于普通文档模板。

### 4.2 白皮书与架构页验收

1. 能讲清楚定位、边界和结构。
2. 有清晰层次，不是大段堆字。
3. 关键图示和结论可以被引用。

### 4.3 模块页验收

1. 每页都要回答“它是什么、解决什么、怎么使用、为什么重要”。
2. 每页都要说明与其他模块的关系。
3. 每页都应尽量有截图或证据支撑。

### 4.4 页面证据页验收

1. 截图标题、说明、路径、操作步骤一致。
2. 同一页面不允许图文不符。
3. 页面证据区应可被售前和交付直接引用。

### 4.5 维护性验收

1. 新增页面不需要推翻既有结构。
2. 新截图替换路径清晰。
3. 状态更新有明确入口。

## 5. 失败信号

如果出现以下情况，说明站点方向发生偏移：

1. 首页看起来像普通文档首页。
2. 智能体运行能力被淹没在普通模块列表里。
3. 站点主卖点无法在真实页面证据中找到对应支撑。
4. 页面越来越多，但读者越来越难找到重点。
5. 视觉越来越炫，但与主产品气质越来越脱节。

## 6. 当前结论

验收标准必须同时覆盖：

1. 认知效果
2. 内容完整性
3. 事实可信度
4. 视觉质量
5. 可持续维护性

后续无论做首页还是做专题页，都应按这五个维度判断是否达标。
