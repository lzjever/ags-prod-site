# AgentSmith 产品网站截图与视觉资产使用规范

更新日期：2026-03-17  
状态：`draft-v0.1`

## 1. 文档目的

本文档用于规范产品网站中截图、局部放大、动画素材和其他视觉资产的使用方式，确保视觉表达高级、可信且可持续维护。

## 2. 基本原则

截图不是装饰素材，而是产品事实和视觉资产。

因此每一张截图都应同时满足：

1. 服务某个明确卖点
2. 与文案叙事一致
3. 不误导实际功能状态
4. 在视觉上有明确主次关系

## 3. 截图分级

### 3.1 Hero 级截图

可用于首页 Hero、专题头图和核心亮点模块。

要求：

1. 信息层次清晰
2. 画面完整度高
3. 能代表关键产品气质
4. 适合裁切后仍保持识别度

当前优先候选：

1. `project-overview.png`
2. `project-chat-session.png`
3. `project-notebook-task-detail.png`
4. `project-files.png`
5. `project-endpoints.png`
6. `project-audit.png`

### 3.2 模块级截图

可用于模块页和亮点区。

要求：

1. 能说明某一模块核心功能
2. 裁切后仍能说明问题
3. 适合搭配一句话价值主张

### 3.3 证据级截图

用于界面与功能说明栏目。

要求：

1. 优先保持页面完整性
2. 说明信息要准确
3. 可以辅以局部高亮，但不能改变实际结构

## 4. 裁切与排版规范

允许的处理方式：

1. 局部裁切
2. 分层拼贴
3. 局部放大
4. 高亮遮罩
5. 关键区域描边或标注
6. 基于同一页面的多焦点拆分排版
7. 同一截图在不同模块中按不同焦点重复使用

不推荐的处理方式：

1. 过度滤镜
2. 改变页面真实色彩倾向
3. 把多个不相关页面硬拼成误导性整体
4. 让截图失真到无法辨认真实产品结构

补充要求：

1. 不应把信息密度过高的整图直接缩小后塞进页面。
2. 如果整图缩小后关键 UI 无法识别，应改用局部放大、裁切重构或多图组合。
3. 任何首页 Hero、专题头图和亮点区图片，都应优先保证焦点清晰，而不是优先保留页面全貌。
4. 设计时应先理解截图中最值得传达的区域，再决定裁切方式。

## 5. 动画使用规范

允许做动画增强的场景：

1. 首页 Hero 截图拼贴
2. 专题页任务流转场
3. 局部高亮切换
4. 分层悬浮和轻微视差
5. 基于截图局部的聚焦切换
6. 多裁切视角之间的渐进过渡

动画必须满足：

1. 不伪造真实功能
2. 不改变截图所表达的产品事实
3. 不依赖动画才能读懂页面内容
4. 静态状态下仍然完整可理解

## 6. 图文对应规则

每个截图使用场景都必须明确对应：

1. 页面标题
2. 页面说明
3. 所属模块
4. 当前状态

禁止出现：

1. 用某个截图去证明另一个页面的功能
2. 用规划中能力的文案描述已实现截图
3. 用已实现截图暗示未实现闭环已经完成

## 7. 视觉增强边界

可以增强：

1. 层次感
2. 焦点引导
3. 版式表现
4. 动效节奏
5. 截图可读性
6. 视觉聚焦和卖点表达

不能增强到：

1. 让截图看起来像另一个产品
2. 掩盖真实界面复杂度
3. 把功能限制隐藏掉

## 8. 维护规则

每个被正式使用的截图应能追溯到：

1. `references/product-docs/latest/` 中的原始文件
2. 对应的 markdown 说明
3. 对应的页面标题和所属模块

如果后续截图更新，应优先检查：

1. 页面标题是否变化
2. 交互结构是否变化
3. 文案说明是否仍然成立

## 9. 当前结论

截图的正确使用方式不是“贴图”，而是“以真实界面为基础进行高质量叙事编排”。

后续所有首页、专题页和证据页设计都应遵循本规范。
