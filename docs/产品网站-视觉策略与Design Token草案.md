# AgentSmith 产品网站视觉策略与 Design Token 草案

更新日期：2026-03-17  
状态：`draft-v0.1`

## 1. 文档目的

本文档用于把产品网站的视觉要求从抽象描述落实为可执行的视觉策略和 design token 草案。

它主要解决四个问题：

1. 站点如何与 `agentsmith` 主产品保持同源协调
2. 站点如何在同源基础上形成更强的产品推广感
3. 后续首页、专题页、文档页和证据页应如何共享同一套视觉系统
4. 后续实现时哪些视觉自由度可以放开，哪些不应漂移

## 2. 总体视觉策略

产品网站的视觉策略不是“重新做一个品牌”，而是：

`以 AgentSmith 主产品的专业深色工具感为基础，叠加更强的叙事纵深、品牌张力和截图编排能力。`

换句话说：

1. `底层气质` 要像主产品
2. `首页冲击力` 要强于主产品后台
3. `文档阅读体验` 要比主产品更克制
4. `专题页表现力` 要比传统 docs site 更有戏剧性

## 3. 视觉关键词

第一版视觉关键词建议固定为：

1. `专业`
2. `深色`
3. `冷静`
4. `可信`
5. `智能感`
6. `结构化`
7. `高密度但不压迫`

禁止偏向以下方向：

1. 轻飘营销风
2. 明亮浅色品牌站
3. 泛紫色 AI 模板风
4. 玻璃拟态堆砌风
5. 赛博噪音过重的概念海报风

## 4. 与主产品的关系

### 4.1 必须继承的部分

产品网站必须继承主产品的以下视觉基因：

1. 深色背景层级
2. 品牌蓝强调色
3. 冷灰文本系统
4. 圆角与边框的克制感
5. 专业工具型排版气质

### 4.2 可以增强的部分

站点允许在以下方面强于主产品后台界面：

1. Hero 区域空间感
2. 截图拼贴与图层关系
3. 叙事型滚动体验
4. AI 渐变与光效表达
5. 专题页的视觉转场

### 4.3 不应偏离的部分

不应偏离以下底层感受：

1. 不能换成暖色主品牌
2. 不能把站点做得像另一个独立品牌
3. 不能让首页视觉与真实产品界面脱节
4. 不能让文档区和营销区看起来像两套站点

## 5. 页面类型的视觉分工

### 5.1 首页

首页负责“建立品牌感与第一印象”，应拥有最强视觉表现力。

适合：

1. 大尺寸标题
2. 截图拼贴 Hero
3. 空间纵深背景
4. 局部动画和滚动联动
5. 高对比模块分区

### 5.2 专题页

专题页负责“放大差异化卖点”，尤其是智能体运行链路专题。

适合：

1. 任务流结构图
2. 节点式视觉节奏
3. 局部截图高亮
4. 分段叙事和视觉转场

### 5.3 文档页

文档页负责“承载系统化说明”，应强调稳定、清晰、可读。

适合：

1. 清晰标题层级
2. 稳定宽度和留白
3. 弱化背景效果
4. 强化内容结构和信息块

### 5.4 证据页

证据页负责“展示真实界面和功能说明”，应突出真实性和可比对性。

适合：

1. 大图展示
2. 局部放大
3. 路径、角色、说明信息条
4. 一致的图文组合模块

## 6. Token 设计原则

站点 token 应遵循以下原则：

1. 以主产品 token 为源头
2. 对首页和专题页进行适度扩展，而不是重建
3. 视觉增强只在高层表现层发生，不改底层身份
4. token 命名应面向用途，而不是面向单页效果

## 8. 背景与光效策略

站点背景建议分为两层：

1. `稳定底层`
   继承主产品深色分层，保证整体同源。
2. `表现层`
   使用极弱的蓝色径向光、顶部雾化高光和局部渐变，增强首页与专题页的空间感。

推荐策略：

1. 文档页背景效果弱
2. 首页背景效果最强
3. 专题页次强
4. 证据页以内容清晰优先

## 9. 排版策略

### 9.1 字体

建议延续主产品字体方向：

1. 正文：`Inter, Noto Sans SC, sans-serif`
2. 代码或技术标签：`JetBrains Mono, monospace`

### 9.2 标题层级建议

1. 首页 Hero 标题：`48-64px`
2. 专题页主标题：`32-40px`
3. 文档页一级标题：`28-32px`
4. 文档页二级标题：`20-24px`
5. 正文：`14-16px`

### 9.3 文本气质

站点文案在视觉上应显得：

1. 清晰
2. 精准
3. 不喊口号
4. 不堆密集灰字墙

## 10. 间距与圆角策略

推荐继承主产品圆角体系，并略做统一：

1. `--site-radius-sm: 8px`
2. `--site-radius-md: 14px`
3. `--site-radius-lg: 18px`
4. `--site-radius-xl: 24px`

间距建议以 8px 为基础单位：

1. `8`
2. `12`
3. `16`
4. `24`
5. `32`
6. `48`
7. `64`

首页和专题页可适当使用更大垂直间距，文档页则保持克制。

## 11. 组件气质建议

### 11.1 按钮

1. 主按钮应使用品牌蓝或高对比深蓝实底
2. 次级按钮应延续深灰描边或弱填充风格
3. 不建议使用高饱和多彩按钮体系

### 11.2 卡片

1. 卡片应以暗面 + 细边框为主
2. 首页和专题页卡片可引入轻微发光边或渐变描边
3. 文档页卡片保持稳定与简洁

### 11.3 标签与状态块

1. 使用冷静的浅蓝、灰蓝、绿色、红色系统
2. 避免糖果色和轻浮配色
3. 状态块必须与 `已实现 / 部分实现 / 规划中` 体系兼容

## 12. 动效策略

动效的目标是：

1. 强化焦点
2. 强化空间关系
3. 强化任务流理解

不应承担：

1. 制造炫技感
2. 代替信息结构
3. 掩盖内容不足

推荐动效强度：

1. 首页：中高
2. 专题页：中
3. 文档页：低
4. 证据页：低到中

## 13. 截图整合策略

截图与 token 系统必须协同：

1. 背景和边框应衬托截图，而不是与截图打架
2. 局部高亮颜色要与品牌蓝体系一致
3. 截图阴影和发光应克制，避免像浮在另一个世界

## 14. 禁止事项

明确禁止：

1. 用紫色替代主产品蓝做全站主色
2. 用过多玻璃拟态和虚化面板覆盖截图
3. 在文档区使用过强动画或过亮背景
4. 让首页和文档页看起来像两个完全不同的品牌

## 15. 当前结论

这份 token 草案的核心不是“定义一套漂亮颜色”，而是确保后续设计始终满足：

1. 与主产品同源
2. 与推广站目标匹配
3. 与文档站阅读体验兼容
4. 与截图和真实界面证据协调

后续进入工程阶段时，应优先把本草案转为 CSS variables 和主题 token。
