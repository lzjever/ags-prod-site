# AgentSmith 产品网站项目宪章

更新日期：2026-03-17  
状态：`draft-v0.1`

## 1. 项目定位

本项目 `ags-prod-site` 不是通用营销官网，也不是简单文档镜像站。

它是 `AgentSmith` 软件项目配套的产品白皮书与渲染网站，目标是把以下三类内容组织成一套统一、可信、可持续演进的产品叙事：

1. `AgentSmith` 的正式产品定位、范围边界与核心价值。
2. 基于真实产品界面、任务链路和治理对象形成的功能说明。
3. 面向外部介绍、内部对齐和交付展示的白皮书式信息架构。

## 2. 项目使命

本项目的使命是：

`以 Next.js + MDX + Fumadocs + 自定义视觉页面搭建一个结构化产品网站，把 AgentSmith 作为“企业级 AI 控制平面 + 通用智能体运行环境”的产品叙事、功能证据和页面素材沉淀为统一站点。`

## 3. 服务对象

本网站主要服务以下读者：

1. 产品负责人：快速理解产品定位、范围边界、路线方向和价值主线。
2. 售前与交付：用统一站点展示产品能力、界面证据和场景价值。
3. 设计与研发：把 PRD、产品截图、代码实现和测试事实串成一致表达。
4. 项目内部成员：避免“文档一套、代码一套、演示一套”的割裂状态。

## 4. 内容事实来源

本项目当前的一级事实来源固定为：

1. `../重构的PRD文档v1`
2. `../agentsmith/artifacts/product-docs`
3. `../agentsmith` 仓库中的代码、测试、合同与开发文档

其中前两者必须先同步到本项目本地副本后再进行网站内容编排。同步脚本：

`./scripts/sync-reference-sources.sh`

## 5. 网站目标

本网站必须同时满足以下目标：

1. 能讲清 AgentSmith 是什么，而不是只罗列页面。
2. 能讲清产品为什么有价值，而不是只堆术语。
3. 能把 `System / Workspace / Project / Use / Govern` 的整体结构讲清楚。
4. 能把 `Chat / Notebook / Files / Agents` 与 `Endpoints / Policy / Usage / Audit` 的关系讲清楚。
5. 能体现通用智能体运行链路，而不把它降格成“一个功能页”。
6. 能为后续持续更新截图、白皮书、功能说明和路线图保留稳定结构。

## 6. 非目标

当前阶段明确不做：

1. 独立的品牌营销官网扩张。
2. 过度包装的宣传文案站，脱离实现事实。
3. 仅按源码目录自动生成的开发文档站。
4. 与 `agentsmith` 主仓库事实脱节的“想象中产品说明”。
5. 在本网站内重新定义一套与 PRD、合同、测试不一致的产品术语。

## 7. 产品网站范围

当前阶段在范围内：

1. 项目介绍与产品摘要。
2. 产品定位、目标、范围边界。
3. 信息架构与核心模块说明。
4. 关键用户故事与任务链路。
5. 技术架构与运行链路高层说明。
6. 基于最新 `product-docs` 的页面级功能说明与截图呈现。
7. 文档来源、版本基线与更新说明。

当前阶段暂不承诺：

1. 多语言版本。
2. 自动从 `agentsmith` 代码实时生成全部页面。
3. 在线交互式产品演示环境。
4. 与主产品站点账号体系打通。

## 8. 核心原则

### 8.1 事实优先

所有站点内容优先以 `PRD + product-docs + 代码/测试/合同` 为依据，不把规划中能力写成已实现。

### 8.2 产品叙事优先

网站组织方式必须优先服务“产品理解”，而不是复刻源码目录或 PRD 文件树。

站点中的市场推广表达必须建立在白皮书级事实之上，而不是与产品说明并行形成两套不一致话术。

### 8.3 统一主线

站点必须始终围绕以下统一表达展开：

`AgentSmith 是企业级 AI 控制平面，同时是通用智能体的易用且安全的运行环境。`

### 8.4 证据化表达

所有关键功能说明，尽量同时具备：

1. 产品语义说明
2. 用户任务说明
3. 页面或截图证据
4. 必要时的代码/合同支撑

### 8.5 可持续维护

后续新增页面、截图、路线图或白皮书章节时，不得破坏站点整体信息架构。

## 9. 站点信息架构原则

站点一级结构建议围绕以下层次展开：

1. 首页 / 产品总述
2. 产品定位与价值
3. 架构与核心对象
4. Use 面能力
5. Govern 面能力
6. 通用智能体运行链路专题
7. 页面证据与截图导览
8. 白皮书 / 路线图 / 版本基线

禁止把站点首页或导航做成：

1. 单纯截图图库
2. PRD 文件树镜像
3. 仅面向工程人员的开发索引

## 10. 内容状态标记

为避免误导，网站内容应统一支持以下状态表达：

1. `已实现`
2. `部分实现`
3. `规划中`

如站点后续出现版本对比、路线图或功能卡片，必须沿用这套状态语义。

## 11. 质量要求

本网站输出至少应满足：

1. 术语一致，不与 PRD 主叙事冲突。
2. 页面结构清晰，便于外部阅读与内部维护。
3. 截图与说明对应，不出现“图文不符”。
4. 白皮书章节与功能说明层次分明，不互相覆盖。
5. 站点可在本地稳定构建、预览和后续增量扩展。

## 12. 当前阶段交付定义

第一阶段交付目标不是立即完成全站，而是建立后续建站基线：

1. 参考资料同步机制
2. 本项目宪章
3. 本项目开发说明书
4. 产品网站内容地图与建站原则

在此基础上，再进入站点工程搭建与内容编排实现。
