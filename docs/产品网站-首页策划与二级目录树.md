# AgentSmith 产品网站首页策划与二级目录树

更新日期：2026-03-17  
状态：`draft-v0.1`

## 1. 文档目的

本文档用于把上一阶段的一级栏目结构继续细化到可执行层，输出：

1. 首页内容策划稿
2. 二级目录树
3. 各栏目的主叙事目标
4. 第一版重点页面之间的跳转关系

它将作为后续站点初始化、导航实现和首页视觉设计的直接依据。

## 2. 首页定位

首页不是“欢迎页”，也不是“文档入口页”。

首页承担四个同时成立的任务：

1. 在 10 秒内让读者知道 AgentSmith 是什么。
2. 在 30 秒内让读者感知它与普通 AI 平台的差异。
3. 在 1 分钟内让读者理解它的两条主线。
4. 在 3 分钟内把读者引导到适合自己的下一层内容。

## 3. 首页核心信息

首页必须稳定传达以下四个核心判断：

1. `AgentSmith 是企业级 AI 控制平面。`
2. `AgentSmith 同时是通用智能体的易用且安全的运行环境。`
3. `它把 Chat、Notebook、Files、Agents 与 Endpoints、Policy、Usage、Audit 统一到同一项目结构下。`
4. `它不是只会接模型，也不是只会跑 Agent，而是把使用、治理、证据和运行环境组织成一个系统。`

## 4. 首页推荐结构

首页推荐由 8 个顺序模块组成。

### 4.1 Hero

目标：

1. 一句话定义产品
2. 建立顶尖、可信、专业的第一印象
3. 立即展示真实产品界面气质

推荐内容：

1. 主标题
2. 副标题
3. 2 个主行动入口
4. 由 `project-overview`、`project-chat-session`、`project-notebook-task-detail`、`project-files` 组成的截图拼贴 Hero

推荐标题方向：

`把企业级 AI 使用、治理与通用智能体运行放进同一个控制平面。`

推荐副标题方向：

`AgentSmith 为企业提供项目级 AI 使用入口、资源治理链路和通用智能体统一运行环境，让 Chat、Notebook、Files、Agents 与审计证据形成闭环。`

推荐 CTA：

1. `阅读产品白皮书`
2. `查看真实界面`

### 4.2 问题定义区

目标：

1. 解释为什么企业真正落地 AI 会变复杂
2. 建立产品存在的必要性

推荐分成两组问题：

1. 企业级 AI 治理问题
2. 通用智能体落地问题

推荐表达方式：

1. 左右双列问题卡片
2. 每列 3 到 4 个关键问题

### 4.3 双主线定位区

目标：

1. 解释 AgentSmith 的双主线定位
2. 避免被误解成单纯 docs、单纯模型面板或单纯 agent 工具

推荐表现：

1. 双列结构
2. 左列 `企业级 AI 控制平面`
3. 右列 `通用智能体运行环境`
4. 中间用共享骨架连接两边

### 4.4 产品结构区

目标：

1. 讲清 `System / Workspace / Project / Use / Govern`
2. 让读者形成清晰心智模型

推荐表现：

1. 分层结构图
2. 滚动联动高亮
3. 模块节点随滚动切换说明

### 4.5 核心能力亮点区

目标：

1. 快速展示四到六个核心亮点
2. 为后续模块页导流

推荐亮点卡：

1. `Chat`
2. `Notebook`
3. `Files`
4. `Agents`
5. `Governance`
6. `Audit Evidence`

每个亮点卡应包含：

1. 一句话价值
2. 局部截图
3. 跳转入口

### 4.6 智能体运行链路区

目标：

1. 突出 AgentSmith 最大差异化
2. 解释 Notebook、Files、Agents、Artifacts、Credential Files 的联动

推荐表现：

1. 横向或纵向任务流动画
2. 关键节点高亮
3. 使用 `project-notebook-task-detail` 与 `project-files` 的截图局部

### 4.7 真实界面证据区

目标：

1. 用真实产品界面建立可信度
2. 让访问者知道这不是概念图

推荐精选页面：

1. 项目总览
2. Chat 多轮会话
3. Notebook 任务详情
4. 文件库管理
5. Endpoint 管理
6. 审计日志

### 4.8 继续阅读区

目标：

1. 根据不同读者角色导向不同栏目
2. 收住首页叙事

推荐分组入口：

1. `给产品与管理者`
2. `给售前与交付`
3. `给设计与研发`

## 5. 首页视觉方向建议

首页配色与主产品保持一致协调，建议：

1. 延续主产品深色背景层次作为基础。
2. 用主品牌蓝和 AI 渐变做重点强调。
3. 用更强的空间纵深、光效和截图编排提升“产品推广感”。
4. 在气质上比主产品更具叙事张力，但不能偏离主产品视觉基因。

推荐站点 token 方向：

1. `bg-base`: 近似 `#121214`
2. `bg-panel`: 近似 `#17181B`
3. `bg-surface`: 近似 `#1F2125`
4. `bg-surface-high`: 近似 `#2B2E34`
5. `accent`: 近似 `#7CA0FF`
6. `text-primary`: 偏冷灰白
7. `text-secondary`: 偏蓝灰

## 6. 二级目录树建议

### 6.1 产品白皮书

建议二级页面：

1. `产品摘要`
2. `定位与范围边界`
3. `核心价值`
4. `成熟度与路线方向`

### 6.2 产品架构

建议二级页面：

1. `整体架构总览`
2. `信息架构与导航结构`
3. `核心对象模型`
4. `关键任务流`
5. `技术架构高层说明`

### 6.3 核心能力

建议二级页面：

1. `Chat`
2. `Notebook`
3. `Files`
4. `Agents`
5. `Governance 总览`
6. `Endpoints`
7. `Resource Policy`
8. `Credentials`
9. `Members`
10. `Usage`
11. `Audit`
12. `Alerts 与 Settings`

### 6.4 智能体运行专题

建议二级页面：

1. `为什么 AgentSmith 不只是一个 Agent UI`
2. `Notebook + Files + Agents 运行链路`
3. `文件持久化与项目资产沉淀`
4. `受控沙箱与托管执行方向`
5. `Skills 与 Credential Files`

### 6.5 界面与功能说明

建议二级页面：

1. `页面索引总览`
2. `工作区与项目`
3. `Chat`
4. `Notebook`
5. `Files`
6. `治理与运营`

建议三级结构如下：

1. `工作区与项目`
   包含工作区选择、工作区登录、工作区项目入口、创建项目对话框、项目总览
2. `Chat`
   包含 Chat 多轮会话
3. `Notebook`
   包含任务列表、创建任务对话框、任务详情
4. `Files`
   包含文件库管理、创建文件库、挂载说明、删除受阻说明
5. `治理与运营`
   包含凭据、Endpoint、Agents、Members、Resource Policy、Audit、Usage、Settings

### 6.6 版本与资料

建议二级页面：

1. `版本基线`
2. `能力状态说明`
3. `资料来源说明`
4. `更新机制`
5. `路线图与变更记录`

## 7. 第一版导航与跳转建议

推荐的关键跳转关系：

1. 首页 Hero 直接跳 `产品白皮书 / 产品摘要`
2. 首页亮点卡分别跳转到对应模块页
3. 首页真实界面证据区直接跳转到对应页面说明
4. `核心能力 / Notebook` 跳转到 `智能体运行专题 / Notebook + Files + Agents 运行链路`
5. `界面与功能说明 / 页面索引总览` 作为截图证据区总入口

## 8. 第一版主导航标签建议

顶部导航建议用短标签：

1. `首页`
2. `白皮书`
3. `架构`
4. `能力`
5. `智能体运行`
6. `界面说明`
7. `版本资料`

如果后续英文版需要同步，这套标签也较容易映射。

## 9. 当前结论

当前已经可以据此进入实际站点搭建阶段。

最重要的执行结论有四条：

1. 首页要按“叙事型产品首页”做，而不是文档首页。
2. 二级目录树已经足够支撑第一版 Fumadocs 内容结构。
3. 站点配色与主产品要同源协调，深色专业感必须保留。
4. 首页、智能体运行专题和页面证据区是第一版最能拉开差异的三个地方。
