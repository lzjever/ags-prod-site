# 10. 已实现能力清单与版本基线

## 10.1 基线结论

基于当前仓库代码与白名单文档，可确认 AgentSmith 已具备一套可运行的 MVP 主产品面。

更准确地说，当前版本基线已经同时具备两条能力主线的骨架：

1. 企业级 AI 治理与控制平面
2. 通用智能体统一操作与受控运行环境

| 能力域 | 关键能力 | 状态 |
|---|---|---|

## 10.3 已经具备雏形的“通用智能体运行环境”能力

虽然这一产品叙事在早期 PRD 中没有被充分突出，但从仓库事实看，以下能力已经形成明显雏形：

| 能力 | 当前判断 | 说明 |
|---|---|---|
| 统一智能体操作界面 | `已实现` | Notebook 已承接 task、inputs、trace、artifacts、SSE 等核心交互，足以作为通用智能体的统一任务入口 |
| 项目级文件输入输出链路 | `已实现` | Files、Notebook Inputs、Artifacts 之间已经形成真实联动 |
| 本地文件系统到文件库挂载桥接 | `已实现` | 已存在 file library mount access 与 credential exchange 能力及说明 |
| 运行上下文注入 | `已实现` | task/workspace/project/run/endpoint/token/inputs/credential files 已进入执行上下文模型 |
| 内置技能自动挂载 | `已实现` | runner 已支持 builtin skills 自动复制到任务工作区 |
| 受控沙箱托管骨架 | `部分实现` | sandbox manager client、pod manager、sandbox starting 事件与 fail-fast 逻辑已存在 |
| 持久化运行文件系统 | `部分实现` | Files、artifact、task workspace 已形成产品方向，但“运行目录与项目文件库自然统一”的闭环仍在增强 |

## 10.4 部分实现能力总表

| 能力域 | 当前状态 | 说明 |
|---|---|---|
| Workspace provisioning backend bootstrap | `部分实现` | 已有状态机、初始化器与 artifact，但不是全部数据面闭环 |
| 数据面 tenant 隔离 | `部分实现` | 已开始对 workspace 私有 collection 使用 tenant prefix，覆盖尚不完整 |
| Internal agent sandbox | `部分实现` | 已有 sandbox manager client / pod manager / UI 状态，但依赖基础设施 |
| Model catalog 管理闭环 | `部分实现` | 已有 bootstrap、store、route 和合同，完整版本化运营仍在增强 |
| Secret handling final posture | `部分实现` | 当前偏 MVP 级实现，后续需硬化 |

## 10.5 可以被视为“当前产品卖点”的能力

1. 系统级 workspace 管理与业务级入口分离。
2. project-scoped 的 AI 使用与治理统一模型。
3. Chat 与 Notebook 共用 endpoint 约束链路。
4. Notebook 作为通用智能体的统一任务界面，显著降低复杂 agent 的使用门槛。
5. Files + Notebook + Agent Trace + Artifacts 的完整链路。
6. 本地文件系统到 AgentSmith 文件库的挂载桥接能力。
7. 外部 agent 与内部 agent 双路径执行能力。
8. user-scoped external connections 与 Feishu skill/tooling 基础。
9. 向受控沙箱、持久化运行文件系统与更广泛 skill 兼容生态演进的清晰结构。

## 10.6 当前版本基线判断

如果用一句产品管理语言总结当前版本基线：

`AgentSmith 已具备面向真实团队试运行的产品骨架与主要模块，而且已经展现出“企业治理平台 + 通用智能体统一运行环境”的双重产品雏形，但控制面深水区和基础设施闭环仍处于持续收敛阶段。`
